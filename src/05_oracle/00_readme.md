# 第 5 章 预言机：Sui 链上数据基础设施

预言机（Oracle）是把链下数据搬上链的桥梁。在 DeFi 中，它最常见的用途是提供价格，但本质上它提供的是一组带信任边界的数据事实：价格由谁发布、多久更新、置信度如何、失效时协议应该继续执行还是暂停。

本章不把预言机当成一个简单接口，而是把它当成协议风险的一部分。读完本章后，你应该能回答三个问题：协议到底消费哪一种价格，价格从链下到链上经过哪些边界，以及当价格过期、偏离或被操纵时，Move 模块应该如何拒绝高风险操作。

## 阅读重点

本章的主线是「价格事实」而不是「项目介绍」。Pyth、Supra、Switchboard 等例子只用于说明不同信任模型和工程接口。真正要带走的是协议侧的防御结构：时间校验、置信区间、偏差阈值、多源仲裁、Fallback 与暂停策略。


## 本章目标

- 区分市场价格、协议价格和可验证价格事实。
- 理解 Pyth、Supra、Switchboard 等预言机在 Sui 上的信任边界。
- 掌握时间、置信度、偏差、多源仲裁四类价格防御。
- 能为借贷、CDP、衍生品设计可替换的预言机插槽。

## 先修知识

- 理解 DEX 价格为什么可能被短时操纵。
- 理解第 2 章的对象分离和事件可观测性。

## 本章小结

预言机的核心不是“读一个数”，而是让协议知道这个数是否足够新、足够可信、足够适合触发高风险操作。所有依赖价格的协议都必须把预言机失效当作正常场景设计。

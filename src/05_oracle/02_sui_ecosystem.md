# 5.2 Sui 预言机生态全景

## Sui 上的预言机版图

```
Sui 预言机生态（2025）：

价格预言机：
  Pyth Network ──── Pull 模式，价格事实，主流
  Supra Oracle ──── Sui 原生，DORA 协议，低延迟
  Switchboard ──── 聚合队列，多数据源

随机数预言机：
  Pyth Entropy ──── commit-reveal VRF
  Supra VRF ────── 原生随机数
  Switchboard VRF ─ 队列式随机数

特色数据预言机：
  Pyth ────────── 40+ 资产类别（外汇、商品、股票）
  Switchboard ──── 自定义数据 Feed
  Supra ────────── 自动化价格流水线 dSTREAM
```

## 各家定位对比

| 维度         | Pyth                     | Supra                   | Switchboard       |
| ------------ | ------------------------ | ----------------------- | ----------------- |
| 数据模型     | 价格事实（Pull）         | 原生预言机（Push/Pull） | 聚合器队列        |
| 延迟         | 低（按需拉取）           | 极低（原生集成）        | 中（队列调度）    |
| 数据源       | 90+ 发布者               | 自有验证者网络          | 任何人可创建 Feed |
| 链上验证     | ECDSA 签名               | DORA 共识               | Oracle 队列共识   |
| Sui 集成深度 | 合约 + SDK               | 原生内置模块            | 合约 + SDK        |
| 随机数       | Entropy（commit-reveal） | VRF（原生）             | VRF（队列）       |
| 适用场景     | 通用 DeFi 价格           | 高频、低延迟场景        | 自定义数据需求    |
| 开源程度     | 开源                     | 开源                    | 开源              |

## Sui 原生模块 vs 外部合约

```
Sui 原生模块：
  └─ sui::clock ── 提供链上时间戳（不是预言机，但常用）
  └─ sui::random ── 提供链上伪随机数（由验证者生成）

外部预言机合约：
  └─ Pyth ── 部署在 Sui 上的独立合约包
  └─ Supra ── 通过原生集成或合约调用
  └─ Switchboard ── 部署在 Sui 上的合约包
```

## 选型决策树

```
你需要什么类型的数据？

├─ 价格数据
│   ├─ 主流资产（BTC/ETH/SOL）→ Pyth 或 Supra 都行
│   ├─ 长尾代币 → Pyth（更多 feeds）或 TWAP 补充
│   ├─ 极低延迟需求 → Supra（原生集成，少一层调用）
│   └─ 需要多源交叉验证 → Pyth + Supra + TWAP
│
├─ 随机数
│   ├─ 游戏/NFT 抽奖 → Supra VRF 或 Pyth Entropy
│   ├─ 需要可验证公平 → Supra VRF（链上证明）
│   └─ 简单随机（不涉及资金）→ sui::random
│
├─ 特色数据（天气、体育等）
│   ├─ 标准金融数据 → Pyth（外汇、商品）
│   └─ 自定义数据 → Switchboard（自建 Feed）
│
└─ 不确定 / 需要灵活切换
    → 设计预言机插槽接口（5.10 节）
```

## 预言机选型的关键指标

```
1. 延迟（Latency）
   价格从产生到链上可用的延迟
   低延迟 = 更准确的价格，但也可能更频繁地需要 gas

2. 覆盖范围（Coverage）
   支持多少种资产/数据类型
   主流预言机覆盖 100+ 交易对

3. 更新频率（Update Frequency）
   价格多久更新一次
   Pull 模式：用户触发更新
   Push 模式：预言机主动推送

4. 安全性（Security）
   数据验证机制、质押要求、历史安全记录

5. 集成复杂度（Integration Complexity）
   PTB 中需要几个步骤
   是否需要额外的链下服务

6. 成本（Cost）
   读取价格消耗的 gas
   某些预言机需要付费订阅
```

## 风险分析

| 风险         | 描述                                             |
| ------------ | ------------------------------------------------ |
| 供应商锁定   | 只用一家预言机，如果它停服或被攻击，协议无法运行 |
| 数据覆盖盲区 | 某些代币可能没有任何预言机提供价格               |
| 集成复杂度   | 不同预言机的接口差异大，切换成本高               |
| 成本差异     | 不同预言机的 gas 成本可能差异很大                |

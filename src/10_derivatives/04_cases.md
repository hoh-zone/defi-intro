# 9.4 Sui 衍生品协议案例与风控

## 订单簿 vs 池/金库模式

| 维度 | 订单簿模式 | 池/金库模式 |
|------|-----------|-------------|
| 价格发现 | 限价单撮合 | AMM 或 oracle 价格 |
| 做市方式 | 用户挂单 | LP 存入资金 |
| 滑点 | 取决于挂单密度 | 取决于池深度 |
| Sui 适用性 | 天然适合（订单=对象） | 同样适合 |
| 代表 | DeepBook Perps | typical vAMM |

## Sui 衍生品协议要点

Sui 上衍生品协议的设计需要特别关注：

1. **共享对象竞争**：PerpMarket 如果是单一共享对象，高并发时 Gas 竞争严重
2. **价格更新频率**：衍生品对价格延迟最敏感
3. **资金费率计算**：需要在链上维护精确的时间记录
4. **保险基金充足性**：极端行情下保险基金可能不够用

## 安全清单

| 检查项 | 说明 |
|--------|------|
| 标记价格来源 | 是否使用多源指数价格？ |
| 资金费率上限 | 是否有资金费率的上限和下限？ |
| 保证金计算 | 是否考虑了未实现亏损？ |
| 清算机制 | 部分清算还是全部清算？清算激励是否充足？ |
| 保险基金 | 保险基金规模相对于未平仓合约的比例？ |
| 暂停机制 | 极端波动时能否暂停开仓？ |
| 最大杠杆 | 是否限制了最大杠杆倍数？ |
| 仓位上限 | 是否有单用户和全局仓位上限？ |

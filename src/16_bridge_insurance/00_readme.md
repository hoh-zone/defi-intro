# 第 16 章 跨链基础设施与链上保险

## 跨链是 DeFi 的"跨国贸易"

一条链就像一个国家：有自己的货币、法律（共识规则）、基础设施。跨链桥就是连接这些国家的贸易路线。

截至 2024 年，跨链桥被攻击造成的损失超过 **20 亿美元**——这比所有其他 DeFi 攻击类别加起来还多。理解跨链桥的安全模型是 DeFi 从业者的必修课。

## 链上保险是 DeFi 的"安全网"

DeFi 协议随时可能被攻击、出 bug、或因经济模型崩溃。链上保险试图回答一个问题：

> **当协议出事时，用户的损失能被弥补吗？**

目前链上保险仍然是一个不成熟的市场——保费定价不准、理赔困难、流动性不足。但理解保险的原理和代码实现，能帮助你评估任何 DeFi 协议的风险敞口。

## 警惕视角

跨链桥的信任假设是最常被低估的风险。用户以为"桥是安全的"，但大多数桥的安全性远低于它们连接的链。一个桥的安全 = min(源链安全性, 目标链安全性, 桥自身安全性)。桥通常是短板。


## 本章目标

- 理解跨链桥的锁铸、销毁释放、消息验证和信任模型。
- 识别桥攻击中的签名、重放、升级和经济激励风险。
- 掌握链上保险池、参数型赔付和裁决机制。
- 理解预测市场为何能成为保险裁决的一种补充机制。

- 理解 DeFi 协议风险可以被转移但不能被消除。

## 本章小结

跨链桥把一个系统的信任假设带到另一个系统，保险则试图给不可避免的风险定价。两者都要求协议明确“谁证明、谁赔付、谁仲裁、谁承担尾部风险”。

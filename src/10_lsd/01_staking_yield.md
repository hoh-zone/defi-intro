# 10.1 质押收益的来源与限制

## 收益从哪来

质押收益来自**网络安全预算**：

$$\text{质押收益} = \text{通胀补贴} + \text{交易手续费}$$

- **通胀补贴**：协议层面的代币发行，分配给验证者和质押者
- **交易手续费**：网络使用者支付的费用，部分分配给质押者

这不是"免费的钱"。它是网络为安全性支付的代价。质押者锁定资本、承担 slashing 风险，换来的是这份收益。

## Sui 的质押机制

Sui 使用委托权益证明（DPoS）。SUI 持有者将代币委托给验证者，获得 epoch 级别的奖励。

### Epoch 机制

Sui 的时间以 **epoch** 为单位（当前约 1 天）。每个 epoch 开始时：

1. **验证者集合确定**：根据质押权重选出当前 epoch 的验证者
2. **奖励分配计算**：上一 epoch 的奖励被计算并分配
3. **质押状态更新**：新的委托和解除质押请求生效

每个 epoch 结束时，验证者获得的奖励按质押比例分配给委托者：

$$\text{委托者奖励} = \text{质押金额} \times \frac{\text{验证者总奖励}}{\text{验证者总质押}} \times (1 - \text{佣金率})$$

佣金率由验证者自行设定（Sui 当前的参考范围为 0%–100%），是验证者提取的收益分成。

### SuiStaked 对象

在 Sui 上，质押后你得到一个 `StakedSui` 对象：

```move
// Sui 系统内置（简化示意）
public struct StakedSui has key, store {
    id: UID,
    pool_id: ID,           // 验证者质押池
    principal: Balance<SUI>, // 本金
    activated_at_epoch: u64,  // 激活 epoch
}
```

关键属性：

- **principal**：你质押的 SUI 数量（不变）
- **激活 epoch 后**：经过至少一个完整 epoch 后开始累积奖励
- **奖励以 SUI 计价**：与验证者表现和全网质押率挂钩

### 收益率的影响因素

| 因素         | 说明                                   |
| ------------ | -------------------------------------- |
| 全网质押率   | 质押率越低，单位质押的通胀奖励越高     |
| 验证者表现   | 节点在线率、提案成功率影响奖励分配     |
| 佣金率       | 验证者抽成，直接影响委托者净收益       |
| Gas 费收入   | 网络活跃度越高，手续费收入越多         |
| 通胀时间表   | 协议设定的代币释放曲线决定长期收益趋势 |

## 原生质押的关键限制

1. **流动性锁定**：质押期间资产被锁定，无法在 DeFi 中使用
2. **解除质押等待**：需要一个完整 epoch 的等待期才能取回 SUI
3. **最低金额**：至少 1 SUI 才能质押
4. **收益币种风险**：收益以 SUI 计价，SUI 价格波动影响实际收益
5. **选择成本**：选错验证者（表现差或佣金高）会降低收益

## 原生质押 vs 流动性质押

| 维度       | 原生质押                     | 流动性质押（LSD）                 |
| ---------- | ---------------------------- | --------------------------------- |
| 流动性     | 锁定                         | LST 可自由转让                    |
| 收益获取   | 自动累积，epoch 级结算       | LST 升值或 rebasing               |
| 退出速度   | 等待一个 epoch               | 在 DEX 卖出 LST（即时但有折价）   |
| 最低金额   | 1 SUI                        | 无限制（通过 DEX 购买碎片化 LST） |
| 额外风险   | 无                           | LST 折价、协议风险、验证者集中    |
| DeFi 组合  | 不参与                       | 可做抵押品、LP、CDP 抵押         |
| 治理参与   | 直接委托投票权               | 通过 LSD 协议间接参与或丧失投票权 |

LSD 的价值主张很简单：**在获得质押收益的同时，保留资产的流动性。** 但这不是免费的午餐——LST 引入了额外的协议风险和折价风险（10.3 和 10.4 会详细讨论）。

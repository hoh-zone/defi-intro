# 22.12 利用率与利率曲线设计

利率曲线是借贷协议最核心的风险参数之一。它决定了借款成本如何随资金需求变化，直接影响协议的流动性和安全性。

## 利用率（Utilization Rate）

```
利用率 U = 总借款 / 总存款

U = 0%   : 所有资金闲置，协议无收入
U = 50%  : 一半资金被借出
U = 80%  : 资金紧张，利率应该快速上升
U = 100% : 资金耗尽，存款人无法提款
```

利用率是连接借款需求和流动性供给的核心指标。

## 利率曲线模型

### 线性模型（最简单）

```
借款利率 = 基础利率 + 利用率 × 利率斜率

例：
  基础利率 = 2%
  利率斜率 = 10%
  当 U = 50% 时，借款利率 = 2% + 50% × 10% = 7%
```

问题：利率上升太慢，在高利用率时无法有效防止流动性枯竭。

### 拐点模型（推荐）

```
             利率
              │          /
              │         /
              │        /
              │       /  ← 拐点后利率快速上升
              │      /
              │     /
              │    /   ← 拐点前利率缓慢上升
              │   /
              │  /
              │ /
              │/
              └──────────── 利用率
              0%    K    100%

K = 拐点（通常在 70-85%）
```

拐点前：利率缓慢上升，鼓励合理借款
拐点后：利率急剧上升，强烈抑制借款，鼓励还款和存款

### 数学表达

```
当 U ≤ K 时:
  借款利率 = R0 + U/K × (R1 - R0)

当 U > K 时:
  借款利率 = R1 + (U - K)/(1 - K) × (R2 - R1)

参数:
  R0 = 基础利率（通常 0-2%）
  R1 = 拐点利率（通常 4-8%）
  R2 = 最大利率（通常 50-300%）
  K  = 拐点利用率（通常 70-85%）
```

## 参数设计的风险考量

### 拐点 K 的选择

| K 值 | 效果 | 风险 |
|------|------|------|
| 过低 (60%) | 利率过早上升，资金效率低 | 借款成本高，用户流失 |
| 适中 (75%) | 平衡资金效率和安全性 | — |
| 过高 (90%) | 资金效率高，但安全缓冲小 | 流动性枯竭风险 |

### 最大利率 R2 的选择

| R2 值 | 效果 | 风险 |
|-------|------|------|
| 过低 (20%) | 借款人在高利用率时仍然愿意借款 | 无法阻止流动性枯竭 |
| 适中 (100%) | 强烈抑制高利用率下的借款 | — |
| 过高 (500%) | 几乎不可能在高利用率下借款 | 可能导致借款人恐慌 |

### 实际案例参考

| 协议 | K | R0 | R1 | R2 |
|------|---|----|----|-----|
| Aave V3 (ETH) | 80% | 0% | 4% | 75% |
| Compound V3 | 90% | 0.5% | 5% | 150% |
| Navi (Sui) | ~80% | ~1% | ~5% | ~100% |

## 存款利率计算

存款利率由借款利率和利用率共同决定：

```
存款利率 = 借款利率 × 利用率 × (1 - 储备因子)

例：
  借款利率 = 5%
  利用率 = 80%
  储备因子 = 10%

  存款利率 = 5% × 80% × 90% = 3.6%
```

储备因子（Reserve Factor）是协议从利息收入中截留的比例，用于构建保险基金。详见 22.14 节。

## 流动性风险管理

利率曲线的核心目的是**管理流动性风险**。评估利率曲线设计的三个关键问题：

### 1. 高利用率场景

当利用率 > 90% 时：
- 存款人能否提款？（如果不能 → 银行挤兑）
- 利率是否足够高以激励还款？
- 是否需要紧急措施？

### 2. 利用率突变场景

当借款需求突然增加（如市场大跌，用户大量借入稳定币）：
- 利率上升速度是否足够快？
- 是否有利率变化速度限制？（防止利率在单个区块内飙升 100 倍）

### 3. 协议收入场景

当利用率长期低迷时：
- 协议是否有足够的收入覆盖运营成本？
- 存款利率是否仍然具有吸引力？

## Sui 协议的特殊考量

Sui 的高 TPS 和低延迟使得利率调整可以更频繁：

- **动态利率更新**：每个 epoch 或更频繁地调整
- **实时利用率监控**：链上即可完成
- **PTB 组合**：可以将利率检查嵌入提款操作中

```move
// 在提款时检查利用率，防止流动性枯竭
fun withdraw(
    market: &mut Market,
    amount: u64,
    ctx: &mut TxContext,
) {
    let new_utilization = calculate_utilization_after_withdrawal(market, amount);
    assert!(new_utilization <= MAX_UTILIZATION, ELiquidityInsufficient);
    // ... 执行提款
}
```

# 22.11 极端行情与清算失败

清算在正常市场条件下运作良好。但在极端行情下，多个因素同时恶化，清算机制可能失效——这正是 DeFi 风险管理最需要关注的时刻。

## 极端行情的特征

| 特征 | 描述 | 发生频率 |
|------|------|---------|
| 价格闪崩 | 几分钟内下跌 20-50% | 每年 2-4 次 |
| 流动性蒸发 | DEX 深度骤降 80%+ | 伴随价格闪崩 |
| Gas 飙升 | 网络拥堵，Gas 价格涨 10-100 倍 | 伴随价格闪崩 |
| 预言机延迟 | 价格更新跟不上实际价格变化 | 伴随价格闪崩 |
| 关联性飙升 | 所有资产同步下跌，分散化失效 | 重大市场事件 |

这些特征通常**同时出现**，形成正反馈循环：

```
价格下跌
  → 大量仓位进入清算
    → 清算出售抵押品
      → 更多抛售压力 → 价格继续下跌
        → 更多仓位需要清算
          → Gas 飙升，清算成本增加
            → 部分清算人退出
              → 清算速度跟不上
                → 坏账累积
```

## Gas 战争

当大量仓位同时需要清算时，清算人竞争提交清算交易，导致 Gas 价格飙升。

```
Gas 战争的三个阶段：

阶段一：正常清算
  Gas 价格: 正常
  清算人: 5 个活跃
  清算速度: 即时
  结果: 所有仓位被及时清算

阶段二：拥堵开始
  Gas 价格: 5-10x 正常
  清算人: 3 个还能盈利
  清算速度: 秒级到分钟级
  结果: 大部分仓位被清算，少数延迟

阶段三：极端拥堵
  Gas 价格: 50-100x 正常
  清算人: 1-2 个有足够资金和 Gas
  清算速度: 分钟级到无法清算
  结果: 大量坏账产生
```

### Gas 战争的影响量化

```
清算盈利性分析：

清算收益 = 抵押品折扣价值 - 债务价值
清算成本 = Gas 费用 + 滑点

正常市场:
  收益 = $1,000 × 8% = $80
  成本 = $2 (Gas) + $5 (滑点) = $7
  净利润 = $73 ✅

极端市场:
  收益 = $1,000 × 8% = $80
  成本 = $100 (Gas) + $50 (滑点) = $150
  净利润 = -$70 ❌ → 清算人退出
```

## 清算拥堵的后果：坏账

当清算失败时，借款人的债务超过抵押品价值——这就是坏账。

```
坏账的产生：

时刻 0:
  抵押品: 10 SUI × $2 = $20
  债务: $15
  健康因子: 1.33 ✅

时刻 1（价格跌到 $1.2）:
  抵押品: 10 SUI × $1.2 = $12
  债务: $15
  健康因子: 0.80 → 应该被清算

时刻 2（清算延迟 5 分钟，价格继续跌到 $0.8）:
  抵押品: 10 SUI × $0.8 = $8
  债务: $15
  坏账 = $15 - $8 = $7 ← 这部分损失由谁承担？
```

### 坏账的分担机制

| 机制 | 承担者 | 说明 |
|------|--------|------|
| 存款人承担 | 协议存款人 | 最常见的默认机制 |
| 储备金承担 | 协议金库 | 需要预先积累储备金 |
| 保险基金 | 保险购买者 | 需要设计激励机制 |
| 代币稀释 | 治理代币持有者 | 通过增发代币弥补 |
| 协议自清算 | 协议自身 | 最后手段 |

## 防御极端清算失败的策略

### 策略一：动态清算参数

```
正常市场: 清算折扣 = 8%, Close Factor = 50%
压力市场: 清算折扣 = 12%, Close Factor = 75%
极端市场: 清算折扣 = 15%, Close Factor = 100%
```

当系统检测到清算压力增大时，自动提高清算奖励和 Close Factor，加速清算进程。

### 策略二：协议自清算

当外部清算人不足时，协议自身执行清算：

### 策略三：熔断机制

当价格变动超过阈值时，暂停新的借款和清算：

```
价格 30 分钟内下跌 > 30%
  → 暂停新借款
  → 暂停非紧急清算
  → 启动治理投票决定后续操作
  → 最多暂停 24 小时（时间锁限制）
```

### 策略四：渐进式清算

不一次性清算整个仓位，而是分批清算：

```
渐进式清算：
  第 1 轮: 清算 20% 债务，给市场时间消化
  第 2 轮（10 分钟后）: 清算 30% 债务
  第 3 轮（10 分钟后）: 清算剩余债务

优点: 减少瞬时抛售压力
缺点: 清算期间价格可能继续下跌
```

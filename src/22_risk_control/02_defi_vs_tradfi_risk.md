# 22.2 DeFi 与传统金融风控的差异

理解 DeFi 风控，最快的方式是与传统金融做对比。差异不是表面的（"去中心化了"），而是根本性的——风险来源、控制手段、治理逻辑都完全不同。

## 身份与信用：无 KYC 的世界

| 维度 | 传统金融 | DeFi |
|------|---------|------|
| 用户识别 | KYC/AML，实名账户 | 匿名地址 |
| 信用评估 | 征信分数、收入证明 | 无信用体系 |
| 贷款审批 | 人工审核 + 自动评分 | 纯抵押驱动，无人工 |
| 违约追索 | 法律诉讼、资产冻结 | 无追索权，依赖清算机制 |
| 风险定价 | 基于信用评分差异化定价 | 统一定价，抵押率唯一变量 |

传统金融通过**信用**管理风险，DeFi 通过**抵押**管理风险。这个根本差异导致：

- DeFi 的资金效率更低（100% 抵押甚至超额抵押）
- DeFi 的风险管理更**机械化**（参数驱动，无弹性）
- DeFi 没有"好人折扣"——所有人都被视为同等风险

## 清算机制：自动 vs 人工

传统金融的清算流程：

```
贷款违约
  → 银行发送催收通知（人工）
  → 协商还款计划（人工）
  → 法院起诉（人工，可能耗时数月）
  → 资产拍卖（人工）
```

DeFi 的清算流程：

```
抵押率低于阈值
  → 任何人都可以触发清算（自动）
  → 清算人获得折扣购买抵押品（激励驱动）
  → 仓位关闭，债务核销（即时）
```

DeFi 的清算速度是优势，但也意味着：
- 没有缓冲时间——价格闪崩时清算可能"来不及"
- 清算依赖外部清算人的**主动参与**——如果没有人清算怎么办？
- 清算本身可能加剧价格下跌——形成**死亡螺旋**（详见 22.17）

## 风控手段对比

| 风控手段 | 传统金融 | DeFi |
|---------|---------|------|
| 风险分散 | 资产组合分散化 | 协议间隔离（但 DeFi 可组合性增加关联风险） |
| 风险转移 | 保险、对冲、信用违约互换 | 链上保险（如 InsurAce）、自保险基金 |
| 风险缓释 | 准备金、资本充足率 | 储备因子、协议金库、紧急提款费 |
| 风险监控 | 风控团队实时监控 | 链上数据监控、预言机告警 |
| 应急处置 | 熔断、停牌、接管 | 紧急暂停、提款限速、治理投票 |
| 监管约束 | 巴塞尔协议、银保监会 | 无外部监管，完全依赖治理 |

## DeFi 独有的风险维度

传统金融不存在、DeFi 独有的风险：

1. **智能合约风险**：代码 bug 可能直接导致资金损失
2. **预言机风险**：链外数据源的可靠性直接影响协议定价
3. **MEV 风险**：交易排序可能被操纵（三明治攻击等）
4. **跨协议传染**：可组合性使得一个协议的风险可以传导到关联协议
5. **治理攻击**：代币投票机制可能被闪电贷操纵
6. **升级风险**：合约升级可能引入新的漏洞

## 对 Sui 协议开发者的启示

Sui 的对象模型在风控方面提供了一些独特优势：

- **对象隔离**：不同用户的仓位是独立对象，一个仓位的清算不影响其他仓位
- **Capability 模式**：权限通过对象所有权强制执行，不存在 Solidity 中的 `tx.origin` 误用
- **PTB（可编程交易块）**：可以在一个交易中组合多个操作，减少 MEV 风险

但这些优势不能替代系统性的风险控制设计。理解传统金融的风控智慧，并适配到 DeFi 的去信任环境中，是每个协议设计者的必修课。

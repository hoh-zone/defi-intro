# 22.7 价格信号：错价风险、操纵模式与防御层次

价格（Price）是 DeFi 协议运转的核心输入。清算、借贷额度、兑换比率、衍生品保证金与 CDP 铸造几乎都依赖价格。**错价就是系统性风险**——本节把「为何重要」「如何被操纵」「如何防御」放在同一框架里，避免在 22.7 / 22.8 两处重复叙述。

> **与第 18 章的分工**：具体攻击路径与案例见 **18.1 预言机操纵**、**18.2 闪电贷攻击**；本节从**风控与参数设计**角度给出可操作的检查项与防御层次。

## 价格在 DeFi 中的角色

```
价格 → 借贷：抵押率与清算触发
价格 → DEX：兑换比率
价格 → 清算：触发条件与抵押品估值
价格 → 衍生品：持仓价值与保证金
价格 → CDP：可铸造稳定币数量
```

## 价格错误的后果分级

| 偏离程度 | 后果               | 影响             |
| -------- | ------------------ | ---------------- |
| < 1%     | 轻微套利机会       | 影响小           |
| 1-5%     | 清算误触发或漏触发 | 部分用户受损     |
| 5-20%    | 大规模错误清算     | 系统性损失       |
| > 20%    | 协议经济模型崩溃   | 可能导致协议破产 |

## 价格使用的三个维度

### 1. 价格来源：数据从哪里来？

| 来源              | 可靠性 | 延迟             | 操纵成本         |
| ----------------- | ------ | ---------------- | ---------------- |
| DEX TWAP          | 中     | 高（需时间窗口） | 中               |
| Chainlink/Pyth 等 | 高     | 低               | 极高             |
| 单一 DEX 现价     | 低     | 极低             | 低（闪电贷即可） |
| 协议自报价格      | 极低   | 无               | 几乎为零         |

### 2. 价格时效：数据有多新鲜？

安全性和时效性之间存在权衡：**TWAP 更安全但更滞后**，即时价格更及时但更容易被操纵（窗口与场景选择见下一节 **22.8**）。

```
                    安全性
                      ↑
   TWAP(30min) ●      │
   TWAP(10min)  ●     │
   即时价格      ●     │
   过期价格       ●    │
                      └──────────→ 时效性
```

### 3. 价格验证：数据是否合理？

```move
fun validate_price(price: u64, reference: u64, max_deviation_bps: u64): bool {
    if (price == 0) return false;
    let lower = reference * (10000 - max_deviation_bps) / 10000;
    let upper = reference * (10000 + max_deviation_bps) / 10000;
    price >= lower && price <= upper
}
```

## 错价时的典型灾难模式（与操纵的衔接）

**借贷（抵押品被高估）**：虚高估值 → 超额借款 → 价格回归 → 坏账风险。

**依赖 DEX 现价的下游协议**：攻击者在低流动性池短时拉盘 → 其他协议误读价格 → 错误清算或错误铸币（与下面「模式一」同一类问题）。

**CDP/稳定币（价格被低估）**：恐慌性清算、抵押品贱卖、脱锚螺旋。

## 闪电贷为何放大操纵（经济学视角）

闪电贷允许**同一交易内**借入巨额资金并归还，使攻击者以**接近零资金占用**撬动池子价格；失败则整笔交易回滚，仅损失 Gas 与少量手续费。因此**任何依赖单一、浅流动性现货报价**的设计都极其危险。更完整的攻击步骤与历史案例见 **18.2**。

## 价格操纵的四种模式（风控视角）

### 模式一：低流动性池操纵

攻击者在大额闪电贷推动下，在低流动性 AMM 中制造巨大价格冲击，再把**被污染的报价**喂给依赖它的模块。**防御**：主价格源不要用浅池现货；必要时用深池 TWAP / 专业预言机。

### 模式二：单区块内多次读价

同一区块内先操纵、再读价、再获利。**防御**：多区块 TWAP、预言机更新节奏与业务读价解耦（详见 **22.8** 与第 **5.9** 节四层防御）。

### 模式三：预言机滞后

链下剧变而预言机仍显示旧价。**防御**：`staleness` 检查、最大有效时长、熔断。

### 模式四：跨协议价格不一致

不同模块使用不同价格源或不同更新频率，产生可套利错价。**防御**：统一主价格模块、聚合策略与偏差阈值全协议一致。

## 操纵成本量化（用于评估「能不能当价格源」）

```
操纵成本 ≈ 在目标流动性环境下推动价格变化 X% 所需资金量

恒定乘积 AMM 粗算：推动价格变化与储备、目标变动比例相关——池子越浅，同等变动所需资金越小。
```

| 操纵成本量级      | 风险 | 建议                     |
| ----------------- | ---- | ------------------------ |
| 极低（如 < $10K） | 🔴   | 不可作为安全价格源       |
| 低                | 🟠   | 仅作辅助，须 TWAP / 多源 |
| 中                | 🟡   | 可配合 TWAP 与偏差限制   |
| 高                | 🟢   | 可作为参考之一           |

## 防御层次（与第 5 章对照）

```
第一层：可靠价格来源（预言机优先，避免单点浅池现货）
第二层：时间维度（TWAP、更新频率、延迟策略）—— 细节见 22.8
第三层：空间维度（多源聚合、截尾/中位数、异常过滤）
第四层：业务侧（单笔/累计限额、熔断、暂停）
```

**实现层**：安全读取模式与示例见第 **5 章 5.9**「安全价格读取函数：四层防御」。

## 价格风险评估清单

1. **单一来源依赖**：主源失效或遭操纵时的影响范围？
2. **操纵成本 vs 攻击收益**：是否可能为正值？
3. **时效**：可接受延迟 1 分钟 / 10 分钟 / 1 小时 时，各自会触发什么业务后果？
4. **极端值**：价格为 0 或跳变 10 倍时，协议是否进入安全失败态（暂停优于错算）？
5. **备用路径**：主源异常时是否降级或停机？

# 第 22 章 DeFi 风险控制全景

前四章（第 17-20 章）从攻击者的视角看 DeFi 安全：什么会出错、如何写安全的代码、如何审计。本章换一个视角——**从系统设计者的角度看风险管理**。

一个 DeFi 协议的长期存活，不只取决于代码没有漏洞，更取决于：

- 风险是否被**识别**和**量化**
- 经济激励是否**对齐**了参与者利益
- 极端情况下系统是否**有足够的缓冲**来吸收冲击

本章不重复攻击细节（那些在第 18 章已详述），而是建立一个**系统级的风险控制框架**，帮助你从"代码安全"走向"协议安全"。

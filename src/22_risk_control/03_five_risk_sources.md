# 22.3 DeFi 风险的五大来源

DeFi 协议面临的风险可以归纳为五个来源。记住这五个来源，就拥有了分析任何 DeFi 协议风险的框架。

## 五大风险来源

```
┌─────────────────────────────────────────────┐
│              DeFi 风险的五大来源              │
├──────────┬──────────┬──────────┬─────────────┤
│  代码风险 │ 价格风险  │ 清算风险 │ 流动性风险   │
│          │          │          │             │
│ 治理风险  │          │          │             │
└──────────┴──────────┴──────────┴─────────────┘
```

### 1. 代码风险

**本质**：代码逻辑与预期行为不一致。

| 子类型   | 描述                           | 典型案例                                      |
| -------- | ------------------------------ | --------------------------------------------- |
| 重入攻击 | 函数在状态更新完成前被重复调用 | 旧版 Solidity 常见，Move 通过资源模型基本消除 |
| 权限错误 | 未授权用户可以调用受限函数     | 缺少 Capability 检查                          |
| 逻辑漏洞 | 业务逻辑的边界情况未处理       | 清算计算中的精度损失                          |
| 升级风险 | 合约升级引入新的漏洞           | 升级后参数范围变化                            |
| 整数溢出 | 数学运算超出类型范围           | 乘法溢出导致数量计算错误                      |

> Sui Move 的类型系统消除了大部分经典代码风险（重入、双花），但逻辑漏洞和权限设计缺陷仍然存在。详见第 21 章。

### 2. 价格风险

**本质**：协议使用的价格数据与真实市场价格偏离。

价格风险的具体表现：

- **预言机延迟**：链上价格更新不及时，协议用过期价格做决策
- **价格操纵**：攻击者通过闪电贷人为影响价格
- **极端波动**：市场价格在短时间内剧烈变化，超过协议的风险参数
- **价差风险**：不同数据源之间的价格差异

```
真实价格: $1,000
预言机价格: $1,200 (被操纵)
差价: 20%

→ 清算判断错误
→ 错误的借贷容量
→ 套利者可以提取价值
```

### 3. 清算风险

**本质**：系统无法及时清算不足额仓位。

清算风险的触发条件：

1. **价格急剧下跌** → 大量仓位同时进入可清算状态
2. **Gas 价格飙升** → 清算交易成本过高，清算人无利可图
3. **流动性不足** → 清算获得的抵押品无法在市场上变现
4. **清算机制设计缺陷** → Close Factor 过低、清算激励不足

清算失败的后果：**坏账**。协议的负债超过资产，存款人无法全额提款。

### 4. 流动性风险

**本质**：用户无法在需要时提取或交易资产。

流动性风险的两个层面：

- **协议层流动性风险**：借贷协议的资金利用率过高，存款人无法提款
- **市场层流动性风险**：DEX 池的深度不足，大额交易造成严重滑点

```
利用率模型：

利用率 U = 借款总额 / 存款总额

U = 0%   → 所有资金可用，无风险
U = 80%  → 资金紧张，提款可能受限
U = 100% → 资金耗尽，提款失败 → 银行挤兑
```

### 5. 治理风险

**本质**：协议的治理机制被滥用或失效。

治理风险的具体形式：

- **闪电贷投票**：借入治理代币来操纵投票结果
- **恶意提案**：通过治理修改协议参数以提取价值
- **权限滥用**：管理员使用紧急权限进行不当操作
- **治理冷漠**：投票参与率过低，少数人控制协议方向

## 风险关联图

五种风险不是独立的——它们相互关联，一种风险的爆发可能触发其他风险：

```
价格下跌 → 清算压力增加 → 市场抛售 → 进一步价格下跌
    ↓                              ↓
预言机延迟                   流动性枯竭
    ↓                              ↓
清算失败 → 坏账产生 → 协议资不抵债 → 治理紧急干预
                                          ↓
                                    治理攻击风险
```

这就是所谓的**风险传染**——一个初始事件可以通过链式反应在多个风险维度上扩散。

## 风险优先级矩阵

不同协议阶段，五种风险的优先级不同：

| 协议阶段 | 最高优先级 | 次高优先级 | 说明                             |
| -------- | ---------- | ---------- | -------------------------------- |
| 开发期   | 代码风险   | —          | 确保代码正确是第一要务           |
| 上线初期 | 代码风险   | 价格风险   | TVL 小，攻击吸引力低但代码风险高 |
| 增长期   | 价格风险   | 流动性风险 | TVL 增长吸引攻击者               |
| 成熟期   | 治理风险   | 系统性风险 | TVL 大，治理价值高               |
| 熊市期   | 流动性风险 | 清算风险   | 价格下跌触发清算和流动性危机     |

本章后续各节将逐一深入每种风险的量化方法和控制手段。

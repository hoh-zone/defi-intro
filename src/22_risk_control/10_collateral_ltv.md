# 22.10 抵押借贷模型与 LTV

抵押借贷是 DeFi 信用体系的基础。理解抵押率设计原理，是管理清算风险的前提。

## LTV（Loan-to-Value）基本模型

LTV 是衡量仓位健康度的核心指标：

```
LTV = 借款价值 / 抵押品价值 × 100%

例：
  抵押 10 SUI（$200/个 = $2,000）
  借款 1,000 USDC

  LTV = 1,000 / 2,000 × 100% = 50%
```

### 关键参数

| 参数 | 含义 | 典型值 |
|------|------|--------|
| 最大 LTV | 允许的最大借款比例 | 70-85% |
| 清算 LTV | 触发清算的阈值 | 80-90% |
| 安全缓冲 | 最大 LTV 与清算 LTV 之差 | 5-15% |

```
LTV 阈值示意：

0%                    最大LTV    清算LTV        100%
 |------- 安全区 -------|--- 缓冲 ---|--- 危险区 ---|
                        75%          85%
```

## 抵押率设计原则

### 原则一：波动性决定最大 LTV

资产的波动性越大，最大 LTV 应该越低：

```
最大 LTV 与资产波动性的关系：

最大 LTV ≈ 1 - (N × 日波动率 × sqrt(清算延迟天数))

其中 N 是安全系数（通常取 3-5）
```

| 资产类型 | 30日波动率 | 建议最大 LTV | 清算 LTV |
|---------|-----------|-------------|---------|
| 稳定币 (USDC/USDT) | < 1% | 85-95% | 90-95% |
| 主流币 (BTC/ETH/SUI) | 3-5% | 70-80% | 80-85% |
| 蓝筹 DeFi 代币 | 5-10% | 50-65% | 65-75% |
| 长尾资产 | > 10% | 30-50% | 50-60% |
| LP 代币 | 取决于组成 | 40-60% | 55-70% |

### 原则二：清算缓冲必须覆盖价格冲击

清算缓冲 = 清算 LTV - 最大 LTV

这个缓冲需要足够大，使得：
1. 价格在正常波动范围内不会触发不必要的清算
2. 价格出现异常波动时，清算人有足够的时间反应

```
缓冲计算示例：

SUI 当前价格: $2.00
最大 LTV: 75%
清算 LTV: 85%

用户借到了最大 LTV：
  抵押品: 100 SUI = $200
  借款: $150 (LTV = 75%)

触发清算的价格：
  $150 / (100 × P) = 85%
  P = $150 / 85 = $1.765

价格需要下跌: ($2.00 - $1.765) / $2.00 = 11.75%

如果 SUI 日波动率 ~5%：
  安全缓冲 ≈ 2.35 个标准差（11.75% / 5%）
  → 大约 99% 的时间内不会触发错误清算
```

### 原则三：不同资产组合的 LTV 计算

当用户用多种资产作为抵押时，LTV 计算需要考虑**组合风险**：

```
方法一：简单加权
  总抵押价值 = Σ (资产数量 × 价格 × 折扣因子)
  总借款价值 = Σ (资产数量 × 价格)
  LTV = 总借款价值 / 总抵押价值

方法二：风险加权（更安全）
  每种资产有不同的最大 LTV
  有效借款 = Σ (借款_i / maxLTV_i)
  有效抵押 = Σ (抵押品_i)
  LTV = 有效借款 / 有效抵押
```

## LP 代币作为抵押品的特殊考虑

LP 代币是 DeFi 中常见的抵押品类型，但有特殊风险：

### 无常损失风险

```
LP 代币价值 = 2 × sqrt(数量_A × 价格_A × 数量_B × 价格_B)

当价格比例变化时：
  如果价格变化 +100%（翻倍），LP 价值变化约 +59%
  而单纯持有资产变化约 +100%

  差额 = 100% - 59% = 41% = 无常损失
```

### LP 抵押品的折扣

| LP 类型 | 建议折扣因子 | 原因 |
|---------|-------------|------|
| 稳定币-稳定币 (USDC-USDT) | 5-10% | 极低无常损失 |
| 主流币-主流币 (ETH-SUI) | 20-35% | 中等无常损失 |
| 主流币-山寨币 | 35-50% | 高无常损失 |
| 集中流动性 (CLMM) LP | 30-50% | 价格区间外 LP 价值归零 |

CLMM（集中流动性）LP 的风险更高，因为当价格超出指定范围时，LP 的全部流动性集中在一侧，实际价值可能远低于面值。

## Sui 上的抵押借贷实现

在 Sui 的对象模型中，仓位是独立的拥有的对象：

```move
// 每个用户的仓位是独立的对象
public struct Position<phantom Collateral> has key {
    id: UID,
    collateral: Balance<Collateral>,  // 抵押品
    debt: Balance<CDP>,               // 债务
}
```

这种设计的优势：
- **清算隔离**：清算一个仓位不影响其他仓位
- **并行处理**：不同用户的操作可以并行执行
- **精确追踪**：每个仓位的状态独立可查

风险控制要点：
1. 始终使用可靠的预言机价格评估抵押品价值
2. 不同类型资产的 LTV 参数应该经过回测验证
3. 参数调整应该通过治理投票，并设有时间锁
4. 紧急情况下应该能够快速调整参数（如通过紧急多签）

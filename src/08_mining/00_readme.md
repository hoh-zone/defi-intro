# 第 8 章 流动性挖矿：激励算法与排放设计

## 流动性挖矿是协议的"用户获取成本"

前七章讲了 DEX、预言机、聚合器、借贷——这些协议的核心问题是：**初始流动性从哪来？**

传统交易所用做市商（MM）提供流动性，做市商赚取价差。DeFi 没有做市商协议，只有算法。流动性挖矿的本质是：**协议用自己的代币购买流动性**。

这听起来简单，但实现中涉及精确的数学和复杂的状态管理：

- **奖励累加器**：如何在用户不断进出时公平分配奖励
- **排放调度**：奖励数量随时间如何衰减
- **权重分配**：多个池子之间如何分配有限的预算
- **Boost 机制**：如何让长期参与者获得更高奖励

每一项都需要精确的链上计算。算错一个系数，奖励就可能被抽干或永远无法领取。

## 警惕视角

流动性挖矿是 DeFi 中最容易产生"表面繁荣"的机制。TVL 暴涨不代表协议健康，可能只是 mercenary capital（雇佣兵资本）在追逐高 APR。本章每一节都会追问：**这个激励对谁有利？对协议有利吗？能持续吗？**

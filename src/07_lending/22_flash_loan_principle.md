# 7.22 Flash Loan 原理

闪电贷允许在无抵押的情况下借入大量资金，只要在同一笔交易内归还。这是 DeFi 独有的金融创新。

## 为什么可以无抵押

```
传统贷款的风险:
  借钱跑路 → 无法追回 → 需要抵押

闪电贷的"魔法":
  借钱 → 使用 → 归还
  全部在同一个原子交易中完成

  如果没归还 → 交易回滚 → 好像什么都没发生

关键: 原子性保证
  → Sui/EVM 上的交易是原子的
  → 要么全部成功，要么全部失败
  → 不存在"借了但不还"的中间状态
```

## 闪电贷的用途

### 1. 套利

```
DEX A: 1 SUI = $2.00
DEX B: 1 SUI = $2.05

闪电贷操作:
  1. 闪电借入 $1,000,000 USDC
  2. 在 DEX A: $1,000,000 → 500,000 SUI
  3. 在 DEX B: 500,000 SUI → $1,025,000
  4. 归还 $1,000,000 + 手续费 $1,000
  5. 利润: $24,000

无闪电贷:
  需要自己有 $1,000,000
  → 门槛极高

有闪电贷:
  零本金套利
  → 只要有利可图就能执行
```

### 2. 抵押品互换

```
Alice 在借贷协议:
  抵押: 100 ETH
  借款: 100,000 USDC

想换成 SUI 做抵押（认为 SUI 会涨）

传统方式:
  1. 准备 100,000 USDC
  2. 还款 → 取回 ETH
  3. 卖掉 ETH 买 SUI
  4. 存入 SUI → 重新借款
  → 需要大量资金 + 多笔交易

闪电贷方式（一笔交易）:
  1. 闪电借入 100,000 USDC
  2. 还款 → 取回 100 ETH
  3. 在 DEX 卖掉 ETH 买 SUI
  4. 存入 SUI → 借出 100,000 USDC
  5. 归还闪电贷 + 手续费
  → 原子化，零资本需求
```

### 3. 自清算

```
Alice 的仓位 HF 接近 1.0:
  → 如果被清算，损失 5-10% 的抵押品

自清算（避免清算罚金）:
  1. 闪电借入 USDC
  2. 自己还款 → 取回全部抵押品
  3. 卖部分抵押品归还闪电贷
  → 损失只有闪电贷手续费（远低于清算罚金）
```

## 闪电贷的风险

```
对协议:
  → 资金安全（原子性保证）
  → 收取手续费
  → 增加协议使用量

对用户:
  → 如果操作失败，只损失 Gas 费
  → 不需要本金

对市场:
  → 加速价格发现
  → 增加市场效率
  → 但也可能用于攻击（如预言机操纵）

手续费:
  → 通常 0.01%-0.3%
  → Aave: 0.05%
  → 我们的 flash_loan 代码: 可配置（默认 0.3%）
```

## 闪电贷 vs 普通贷款

```
维度         │ 闪电贷          │ 普通贷款
────────────┼─────────────────┼──────────────
抵押         │ 不需要          │ 需要超额抵押
金额上限     │ 池子总量        │ 取决于抵押品
时间限制     │ 同一交易内      │ 无限制
利息         │ 固定手续费      │ 浮动利率
风险         │ 极低（原子性）  │ 清算风险
使用门槛     │ 需要编程能力    │ 任何人
用途         │ 套利/清算/策略  │ 资金需求
```

## 总结

```
闪电贷核心原理:
  原子交易保证同一笔交易内归还
  → 不需要抵押
  → 不需要信任

三大用途:
  套利: 零资本捕获价差
  抵押品互换: 原子化仓位调整
  自清算: 避免清算罚金

风险控制:
  原子性 + 手续费
  → 协议安全，用户有成本
```

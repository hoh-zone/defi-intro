# 7.16 Jump Rate Model（核心）

Jump Rate Model（跳跃利率模型）是 Aave 和 Compound 采用的行业标准。它在线性模型的基础上增加了一个"拐点"。

## 核心思想

```
问题: 线性模型在高利用率时保护不足
方案: 在某个利用率（kink）处"跳跃"增加斜率

两段式设计:
  U ≤ kink:  rate = base + U × slope1      （温和增长）
  U > kink:  rate = base + kink×slope1      （跳跃增长）
             + (U - kink) × slope1 × slope2

参数:
  base_rate:  基础利率 (U=0 时的利率)
  multiplier: slope1，拐点以下的斜率
  kink:       拐点利用率 (如 80%)
  jump:       slope2，拐点以上的额外斜率倍数
```

## 利率曲线

```
利率 (%)
  30 ┤                          ╱
  25 ┤                        ╱
  20 ┤                      ╱  ← 拐点以上: 陡峭
  15 ┤                    ╱
  10 ┤            -------╱     ← 拐点 (kink=80%)
   8 ┤          /
   6 ┤        /
   4 ┤      /                   ← 拐点以下: 平缓
   2 ●----/
   0 └──┬──┬──┬──┬──┬──┬──┬──→ U (%)
      0  10 20 30 50 70 80 90 100

拐点以下: 温和增长 → 鼓励借款
拐点以上: 急剧增长 → 保护流动性
```

## 公式推导

```
设:
  U = 利用率
  R_b = base_rate
  M_1 = multiplier (slope1)
  U_k = kink
  J = jump_multiplier (slope2)

情况 1: U ≤ U_k
  rate = R_b + U × M_1

  示例 (U=50%, R_b=2%, M_1=10%):
  rate = 2% + 50% × 10% = 7%

情况 2: U > U_k
  rate = R_b + U_k × M_1 + (U - U_k) × M_1 × J

  拐点处的利率:
  rate_at_kink = R_b + U_k × M_1

  超过拐点的额外利率:
  excess = (U - U_k) × M_1 × J

  示例 (U=90%, R_b=2%, M_1=10%, U_k=80%, J=5x):
  rate_at_kink = 2% + 80% × 10% = 10%
  excess = (90% - 80%) × 10% × 5 = 5%
  rate = 10% + 5% = 15%
```

## lending_market 的参数

```
默认参数 (BPS):
  base_rate_bps = 200       (2%)
  kink_bps = 8000           (80%)
  multiplier_bps = 1000     (10%)
  jump_multiplier_bps = 5000 (5x)

不同利用率下的利率:
  U=0%:   rate = 200 = 2.0%
  U=20%:  rate = 200 + 2000×1000/10000 = 400 = 4.0%
  U=40%:  rate = 200 + 4000×1000/10000 = 600 = 6.0%
  U=60%:  rate = 200 + 6000×1000/10000 = 800 = 8.0%
  U=80%:  rate = 200 + 8000×1000/10000 = 1000 = 10.0% ← kink
  U=85%:  rate = 1000 + 500×1000×5000/10000^2 = 1025 = 10.25%
  U=90%:  rate = 1000 + 1000×1000×5000/10000^2 = 1050 = 10.5%
  U=95%:  rate = 1000 + 1500×1000×5000/10000^2 = 1075 = 10.75%
  U=100%: rate = 1000 + 2000×1000×5000/10000^2 = 1100 = 11.0%
```

## 为什么拐点设在 80%

```
80% 的含义:
  → 80% 的资金被借出
  → 20% 的流动性缓冲
  → 足够存款人日常取款

低于 80%:
  → 充足的流动性
  → 不需要高利率来抑制借款
  → 利率温和增长（鼓励合理借款）

高于 80%:
  → 流动性紧张
  → 需要高利率来:
    1. 抑制新的借款
    2. 鼓励现有借款人还款
    3. 吸引新的存款人
  → 利率急剧增长（保护流动性）
```

## 不同资产的参数选择

```
资产类型     │ base │ kink │ slope1 │ jump │ 原因
────────────┼──────┼──────┼────────┼──────┼──────────
稳定币       │ 0-2% │ 80%  │ 4-8%   │ 3-5x │ 低风险，高需求
蓝筹(ETH/SUI)│ 2-3% │ 70%  │ 8-12%  │ 5-7x │ 中风险
长尾资产     │ 5-10%│ 60%  │ 15-25% │ 7-10x│ 高风险，需高利率补偿

利率越高:
  → 借款成本越高
  → 存款人收益越高
  → 风险补偿越大
```

## 总结

```
Jump Rate Model 核心公式:
  U ≤ kink: rate = base + U × multiplier
  U > kink: rate = base + kink × multiplier + (U-kink) × multiplier × jump

关键参数:
  kink: 拐点（通常 70-80%）
  multiplier: 拐点以下斜率
  jump_multiplier: 拐点以上跳跃倍数

这是 Aave/Compound 的标准利率模型
也是 lending_market 代码中实现的模型
```

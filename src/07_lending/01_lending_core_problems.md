# 7.1 借贷协议的核心问题

借贷是 DeFi 最核心的金融原语之一。本节分析链上借贷需要解决的核心问题。

## 为什么需要超额抵押

```
TradFi 借贷:
  银行借钱给你 → 基于你的信用评分、收入证明
  如果不还 → 法律追偿

DeFi 借贷:
  协议借钱给你 → 没有身份系统、没有法律追偿
  如果不还 → ??? 没有办法追回

解决方案: 超额抵押（Over-Collateralization）
  你必须存入价值 > 借款金额的抵押品
  如果抵押品价值下降 → 自动清算

为什么不能欠款跑路:
  抵押品锁定在链上合约中
  任何人都可以触发清算（经济激励）
  去中心化的"追偿"机制
```

### 超额抵押的数学约束

```
存入: 100 SUI（价值 $200）
最多借出: $200 × 75% = $150（LTV 上限）

LTV (Loan To Value) = 借款价值 / 抵押品价值
  LTV = 75% → 安全
  LTV = 80% → 到达清算阈值
  LTV > 80% → 被清算

为什么要留 5% 的安全缓冲?
  → 价格可能在区块间剧烈波动
  → 清算人需要时间反应
  → 没有缓冲 = 坏账风险
```

## 借贷如何创造流动性

```
没有借贷时:
  Alice 有 1000 SUI 闲置 → 0% 收益
  Bob 需要 500 SUI 做交易 → 无法获得

有借贷时:
  Alice 存入 1000 SUI → 获得 3% 年化收益
  Bob 用 750 SUI 做抵押 → 借出 500 USDC
  协议赚取利差 → 可持续运营

三方受益:
  存款人: 闲置资产产生收益
  借款人: 不卖资产就能获得流动资金
  协议: 赚取利差（spread）
```

## 三个核心角色

### Lender / Supplier

```
目标: 让闲置资产产生收益

行为:
  1. 存入资产到借贷池
  2. 获得存款凭证（Share Token）
  3. 随时可以取回（前提是池子有流动性）

风险:
  → 智能合约风险（黑客攻击）
  → 流动性风险（所有资金被借出时无法取回）
  → 利率变动（收益不固定）

收益来源:
  → 借款人支付的利息
  → 部分协议代币激励
```

### Borrower

```
目标: 不卖出持有资产的情况下获得流动资金

行为:
  1. 存入抵押品
  2. 借出其他资产
  3. 使用借来的资产（交易、投资等）
  4. 偿还借款 + 利息
  5. 取回抵押品

风险:
  → 清算风险（抵押品价格下跌）
  → 利率风险（浮动利率可能升高）

常见用例:
  → 看涨 SUI 但需要 USDC → 抵押 SUI 借 USDC
  → 杠杆做多 → 借出后买入更多 SUI 再抵押
  → 套利 → 借出资产进行跨 DEX 套利
```

### Liquidator

```
目标: 维护协议安全，赚取清算奖励

行为:
  1. 监控所有借款仓位
  2. 发现 Health Factor < 1 的仓位
  3. 替借款人偿还部分/全部债务
  4. 获得折扣价的抵押品作为奖励

经济激励:
  → 清算奖励 = 5-10% 的额外抵押品
  → 这是去中心化的"追债"机制
  → 清算人竞争 → 确保清算及时执行

清算人的工具:
  → 自动化机器人（监控链上状态）
  → 闪电贷（零资本清算）
  → MEV 搜索者（优先执行）
```

## DeFi 借贷 vs TradFi 借贷

```
维度       │ TradFi           │ DeFi
──────────┼──────────────────┼──────────────────
抵押要求   │ 信用评分或抵押    │ 必须超额抵押
审批       │ 需要审核          │ 无需审批（链上自动）
利率       │ 固定为主          │ 浮动（随供需变化）
清算       │ 法院/催收        │ 自动化（任何人可触发）
速度       │ 天级              │ 秒级
地理限制   │ 有                │ 无
金额上限   │ 视信用额度        │ 仅受流动性限制
透明度     │ 低                │ 完全透明
```

## 总结

```
借贷协议的核心问题:
  1. 如何在无信任环境下保证还款？→ 超额抵押
  2. 如何为存款人和借款人匹配？→ 利率模型
  3. 如何处理抵押品价值波动？→ 清算机制
  4. 如何确保系统安全？→ Health Factor + 风险参数

这三个角色（Supplier, Borrower, Liquidator）的经济激励
构成了借贷协议的"三位一体"安全保障
```

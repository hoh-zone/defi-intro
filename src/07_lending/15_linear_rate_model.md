# 7.15 线性利率模型

最简单的利率模型——线性模型，是理解利率设计的起点。

## 模型公式

```
borrow_rate = base_rate + U × slope

其中:
  base_rate: 基础利率（U=0 时的利率）
  U: 利用率 (0-1)
  slope: 利率斜率

示例:
  base_rate = 2%
  slope = 10%

  U=0%:   rate = 2% + 0% × 10% = 2.0%
  U=25%:  rate = 2% + 25% × 10% = 4.5%
  U=50%:  rate = 2% + 50% × 10% = 7.0%
  U=75%:  rate = 2% + 75% × 10% = 9.5%
  U=100%: rate = 2% + 100% × 10% = 12.0%
```

## 优点

```
1. 简单易懂
   → 只有两个参数
   → 利率行为可预测

2. 实现简单
   → 一行公式
   → Gas 成本最低

3. 适合低波动资产
   → 稳定币市场（USDC/USDT）
   → 利用率通常稳定
```

## 问题

```
问题 1: 无法保护流动性
  当 U→100%:
    rate = 12%（不够高！）
    → 借款人没有强烈动力还款
    → 存款人无法取款
    → 可能导致流动性危机

问题 2: 缺乏弹性
  利率线性增长
  → 没有对高利用率区域的"惩罚"
  → 无法快速调节极端情况

问题 3: 存款利率可能为零
  supply_rate = U × borrow_rate × (1 - reserve)
  U=0: supply_rate = 0
  → 存款人没有任何收益
  → 没有动力提供流动性

对比:
  线性模型: "一视同仁"，低利用率和高利用率用同样的斜率
  现实需求: 高利用率时需要"紧急刹车"
```

## Move 实现

```move
// 线性利率模型
public fun linear_rate(
    base_rate_bps: u64,
    slope_bps: u64,
    utilization_bps: u64,
): u64 {
    // rate = base + U × slope / 10000
    base_rate_bps + utilization_bps * slope_bps / 10000
}
```

```
BPS 计算:
  base_rate_bps = 200  (2%)
  slope_bps = 1000     (10%)
  utilization_bps = 5000 (50%)

  rate = 200 + 5000 × 1000 / 10000 = 200 + 500 = 700 bps = 7%
```

## 线性模型的适用场景

```
适合:
  → 教学演示（理解利率基本概念）
  → 稳定币市场（利用率波动小）
  → 小型协议（简单可靠）

不适合:
  → 主流资产市场（SUI/ETH/USDC）
  → 高波动环境
  → 需要精细化管理的协议

改进方向:
  → 在高利用率区域加大斜率
  → 这就是 Jump Rate Model 的核心思想
```

## 总结

```
线性利率模型:
  rate = base_rate + U × slope

优点: 简单、可预测
缺点: 高利用率保护不足

这是利率模型的起点
下一节的 Jump Rate Model 是它的改进版
```

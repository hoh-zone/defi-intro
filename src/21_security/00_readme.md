# 第 21 章 Move 与 DeFi 安全实践

## 安全是一个动词

前四章（18-20）讲了攻击模式、工程方法和审计流程。本章回到代码本身：Move 语言的安全模型提供了什么保护，它无法阻止什么，以及你应该如何利用语言特性将风险降到最低。

Move 并非"天然安全"——它只是把 EVM 世界的某些整类漏洞变成了不可能，同时引入了新的陷阱。理解这些边界，是写出可信代码的前提。

本章从类型系统出发，经过对象权限、整数安全、资产处理、密钥管理、接口鉴权，一直到升级安全和工具链，形成一份完整的安全实践手册。每一节都包含可直接使用的 Move 代码。


## 本章目标

- 理解 Move 类型系统、Capability、整数精度、资产处理和公共接口安全。
- 掌握多签、时间锁、升级和紧急响应的 Move 设计模式。
- 能识别 Coin/Balance、共享对象和 public 函数中的常见安全误区。
- 形成上线前 Move 安全检查清单。

## 先修知识

- 熟悉 Sui Move 基础语法和 DeFi 资金流。
- 读过攻击篇和工程化章节，理解安全需求来源。

## 本章小结

Move 给 DeFi 安全提供了更强的类型基础，但安全结果仍取决于业务边界、权限设计、精度处理和接口暴露。类型系统是起点，不是终点。

## 练习题

1. 把一个 `ctx.sender()` 管理员检查改成 Capability 参数。
2. 设计一个防止错误 Coin 类型混入的函数签名。
3. 说明整数除法截断可能如何影响手续费或份额。
4. 列出发布前必须手动检查的五个 public 函数。

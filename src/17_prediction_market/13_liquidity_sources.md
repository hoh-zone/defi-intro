# 17.13 市场流动性来源

流动性决定了价格是否有意义。没有流动性的预测市场，价格只是噪声。

## LMSR 的「内置」流动性

```
传统订单簿:
  需要有人挂买单和卖单 → 如果没人挂，就无法成交
  冷启动问题严重：新市场没人愿意先挂单

LMSR:
  成本函数连续 → 任意数量的买卖都有报价
  不需要对手方 → 合约本身就是做市商
  代价：协议承担最坏损失（≈ b × ln n）

┌──────────────┐       cost = ΔC        ┌──────────────┐
│   Trader     │ ──────────────────→    │   Market     │
│              │ ←──────────────────    │ (LMSR)       │
│              │       shares           │  vault: 金库 │
└──────────────┘                        └──────────────┘

没有对手方 Trader → LMSR 自己就是对手方
资金来源 → vault（seed + 历史交易收入）
```

## 三种流动性模式

### 模式 1：纯 LMSR（本章实现）

```
资金来源: 创建者的 initial_seed
优点: 最简单、冷启动友好
缺点: 资金效率低（seed 被锁定），最坏损失完全由创建者承担

适用: 教学、小型社区市场、低流动性长尾事件
```

### 模式 2：LMSR + 外部 LP

```
资金来源: initial_seed + 外部 LP 注入

流动性池:
  LP 向 vault 存入额外资金
  LP 按份额分享手续费
  LP 在市场结算后按比例取回剩余

优点: 风险分散、更深的流动性
缺点: 需要设计 LP Token + 优先级规则

  seed         LP 注入
   ↓              ↓
┌──────────────────────┐
│         vault        │
│                      │
│  seed_share  lp_share│
└──────────────────────┘
        ↓           ↓
  最坏损失由     LP 按比例
  seed 先吸收   分担剩余
```

### 模式 3：订单簿（Polymarket 等商业系统）

```
资金来源: 专业做市商挂单

  Trader A (买 YES @0.60)
          ↓
  ┌────────────────────┐
  │    匹配引擎         │   ← 通常在链下
  │  (order matching)   │
  └────────┬───────────┘
           │
  ┌────────┴───────────┐
  │    结算合约         │   ← 在链上
  │  (CTF settlement)   │
  └────────────────────┘
          ↑
  Trader B (卖 YES @0.60)

优点: 价格发现效率高、专业做市商提供紧价差
缺点: 链下部分引入中心化风险、冷启动依赖做市商

Polymarket 实际使用的是链下订单簿 + 链上 CTF 结算
→ 不是纯 LMSR → 17.23 详述
```

## 流动性模式对比

| 维度 | 纯 LMSR | LMSR + LP | 订单簿 |
|------|---------|-----------|--------|
| 冷启动 | 容易（有 seed 即可） | 中等 | 困难（需要做市商） |
| 资金效率 | 低 | 中 | 高 |
| 价格发现 | 中等 | 中等 | 好 |
| 去中心化 | 完全链上 | 完全链上 | 通常混合 |
| 实现复杂度 | 低 | 中 | 高 |
| 最坏损失 | 创建者承担 | LP 分担 | 做市商承担 |

## 手续费如何影响流动性

```
fee_bps = 200（2%）

用户买 100 份 YES:
  LMSR 成本: 60 USDC
  手续费: 60 × 2% = 1.2 USDC
  总支付: 61.2 USDC

手续费去了哪里:
  → 进入 vault（教学版）
  → 或分账给协议 + LP（生产版）

手续费过高 → 交易者不愿来 → 价格信息少 → 预测市场失去意义
手续费过低 → 协议亏损 → 不可持续

建议范围:
  长期市场（>30天）: 0.5-1%
  短期市场（<7天）: 1-3%
  高流动性热门市场: 0.1-0.5%
```

## 自检

1. 为什么 LMSR 不需要 LP Token？（答：做市损失由 seed 内部吸收，不需要追踪外部 LP 份额）
2. 如果 seed = 0 且 fee = 0，LMSR 市场能运作吗？（答：可以启动但无法覆盖最坏损失，claim 时可能金库不足）

# 17.18 流动性参数 b

\(b\) 是 LMSR 唯一的可调参数，但它同时控制三件事：**滑点**、**最坏损失**和**价格灵敏度**。理解 \(b\) 的工程含义，是从「能跑」到「能用」的关键一步。

## b 控制什么

```
LMSR 成本函数: C(q) = b × ln(Σ exp(q_i / b))
边际价格:       p_i  = exp(q_i / b) / Σ exp(q_j / b)

b 出现在分母: q_i / b
→ b 越大，q 对价格的影响越小
→ b 越小，q 对价格的影响越大
```

### 三个维度的影响

```
维度 1 — 滑点（Price Impact）:
  交易量固定时:
    小 b → 价格移动大 → 滑点大
    大 b → 价格移动小 → 滑点小

  数值: 买 100 YES（q_yes: 0 → 100, q_no = 0）
    b = 100:  p_YES 从 0.50 → 0.73   成本 = 73.1
    b = 500:  p_YES 从 0.50 → 0.55   成本 = 52.5
    b = 1000: p_YES 从 0.50 → 0.525  成本 = 51.3
    b = 5000: p_YES 从 0.50 → 0.505  成本 = 50.3

维度 2 — 最坏损失:
  做市方最大亏损 ≈ b × ln(n)
  二元市场: ≈ b × 0.693
  三结果: ≈ b × 1.099
  → b 是资本金需求的直接乘数

维度 3 — 信息灵敏度:
  价格反映「市场共识」的速度取决于 b
  小 b → 少量交易就能大幅移动价格 → 灵敏但易被操纵
  大 b → 需要大量交易才能移动价格 → 稳定但反应慢
```

## 实际选参方法

### 方法 1：基于预期成交量

```
核心思路: 让「预期总成交量」导致 p 从 0.5 移动到约 0.75

公式推导:
  p_YES = exp(q_yes / b) / (exp(q_yes / b) + 1)

  想让 p_YES = 0.75 时 q_yes = Q_expected:
    0.75 = exp(Q / b) / (exp(Q / b) + 1)
    exp(Q / b) = 3
    Q / b = ln(3) ≈ 1.099
    b = Q_expected / 1.099

  例如:
    预期总成交量 = 10,000 USDC
    b = 10,000 / 1.099 ≈ 9,100
    最坏损失 ≈ 9,100 × 0.693 ≈ 6,306 USDC
    → 需要 seed ≥ 6,306 USDC
```

### 方法 2：基于可承受损失

```
核心思路: 创建者能承受多少最坏损失

  seed = 1,000 USDC
  最坏损失 ≤ seed → b × 0.693 ≤ 1,000
  → b ≤ 1,443

  取 b = 1,000（留安全余量）
  → 最坏损失 ≈ 693 USDC
  → 可承受 ✅
```

### 方法 3：基于目标滑点

```
核心思路: 让「典型交易」的滑点在可接受范围内

  典型交易 = 50 USDC 买 YES
  可接受滑点 = 1%

  滑点 ≈ shares / (2 × b)（线性近似，小交易时成立）
  0.01 = 50 / (2 × b)
  b = 50 / 0.02 = 2,500

  验证:
    买 50 YES (b=2500): 成本 = C(50,0,2500) - C(0,0,2500)
    ≈ 2500 × ln(e^0.02 + 1) - 2500 × ln(2)
    ≈ 2500 × 0.703 - 2500 × 0.693 = 25.5 USDC
    但如果无滑点应该是 50 × 0.50 = 25.0 USDC
    滑点 = (25.5 - 25.0) / 25.0 = 2% → 接近目标
```

## b 与操纵的关系

```
攻击场景: 操纵者想把 p_YES 从 0.5 推到 0.9

  p_YES = 0.9 时: exp(q_yes/b) = 9 × exp(q_no/b)
  → q_yes - q_no = b × ln(9) ≈ b × 2.197

  b = 100:   需要 q_yes - q_no = 220   成本 ≈ 153 USDC
  b = 1000:  需要 q_yes - q_no = 2197  成本 ≈ 1534 USDC
  b = 10000: 需要 q_yes - q_no = 21972 成本 ≈ 15342 USDC

结论:
  b 越大 → 操纵越贵 → 市场越安全
  但 b 越大 → seed 越大 → 创建门槛越高

安全 vs 效率的 tradeoff:
  社区小市场 b = 100-500 → 便宜但容易操纵
  专业市场 b = 5000-50000 → 安全但门槛高
```

## 动态 b 的可能性

```
静态 b（本章实现）:
  创建时设定，之后不变
  简单、可预测、容易审计

动态 b（高级方案）:
  随时间衰减: b(t) = b_0 × (1 - t/T)
  → 接近截止时滑点增大
  → 鼓励早期交易
  → 实现复杂度上升

  随交易量调整: b 根据历史成交量自适应
  → 高流动性时自动降低 b（减少资金占用）
  → 低流动性时自动提高 b（保证报价）
  → 引入新的攻击面（操纵 b）

教学版不实现动态 b，但读者应知道这是一个活跃的研究方向。
```

## 参数选择检查表

```
上线前必须回答:
  ☐ seed 是否 >= b × ln(n) × 1.2（留 20% 安全余量）?
  ☐ 目标市场的预期单笔交易量是多少？滑点是否可接受？
  ☐ 操纵价格到 0.9 需要多少资金？是否足以阻吓？
  ☐ 如果所有交易者都正确预测，seed 够赔吗？
  ☐ b 设定后是否可修改？如果可修改，谁有权限？
```

# 17.15 为什么需要 AMM

## 预测市场的流动性困境

```
传统股票市场:
  Apple 股票 → 每天数百万笔交易
  买卖价差通常 < 0.01%
  → 不需要担心流动性

预测市场:
  "2025年美联储降息次数 >= 3？"
  → 全世界可能只有几千人关注这个命题
  → 每天可能只有几十笔交易
  → 用订单簿的话，价差可能 > 10%

这就是预测市场的核心矛盾:
  有价值的命题 ≠ 有流动性的市场
  「长尾事件」恰好最需要价格发现，但流动性最差
```

## 订单簿 vs AMM 在预测市场中的表现

```
场景: 新市场「SUI 年底突破 $10？」，刚上线

订单簿:
  买单: 0.30 × 100 USDC（一个人挂了）
  卖单: 0.70 × 50 USDC（另一个人挂了）
  价差: 0.70 - 0.30 = 0.40（40%！）
  → 没人愿意以这个价差交易
  → 市场「死了」

LMSR (b = 500):
  初始 q_yes = q_no = 0 → p_YES = 0.50
  用户想买 100 YES:
    成本 = C(100, 0, 500) - C(0, 0, 500)
         = 500 × ln(e^0.2 + 1) - 500 × ln(2)
         ≈ 500 × 0.798 - 500 × 0.693
         = 52.5 USDC
  → 价格从 0.50 移动到 e^0.2/(e^0.2+1) ≈ 0.55
  → 始终有报价，没有「死市场」
```

## LMSR 解决的三个问题

### 问题 1：冷启动

```
订单簿冷启动:
  没有交易者 → 没有挂单 → 没有价格
  没有价格 → 没有参考 → 没有交易者
  → 死循环

LMSR 冷启动:
  创建市场 + seed → 立即有报价 (p = 0.5)
  第一个交易者看到报价 → 直接交易
  → 没有死循环
```

### 问题 2：连续报价

```
订单簿:
  只在挂单价位上有报价
  价位之间没有成交可能

LMSR:
  C(q) 是连续可微函数
  任意小的 Δq 都有对应的成本
  → 无论买多小的量，都有报价
```

### 问题 3：确定性定价

```
订单簿:
  成交价取决于对手方挂单 → 不确定
  可能被 MEV 前跑 → 实际成交价偏离预期

LMSR:
  ΔC = C(q') - C(q) → 完全确定
  给定 q 和 shares，成本是纯函数
  → 链上可验证，不依赖对手方
```

## LMSR 的代价

```
代价 1 — 最大损失:
  LMSR 做市方承担最坏损失 ≈ b × ln(n)
  n = 2 时: 最坏损失 ≈ 0.693 × b
  → 需要预留资金覆盖

代价 2 — 参数敏感:
  b 太大: 需要太多资金，但滑点小
  b 太小: 资金需求少，但滑点大
  → 没有「万能」的 b

代价 3 — Gas 成本:
  链上计算 exp/ln 需要泰勒级数
  比简单的 x × y = k 消耗更多 Gas
  → 需要工程优化

代价 4 — 没有价格改善:
  订单簿中，限价单可以提供更优价格
  LMSR 的价格完全由公式决定
  → 对大单不如订单簿友好
```

## LMSR vs CPMM 对比

| 维度 | LMSR | CPMM (x·y=k) |
|------|------|--------------|
| 设计目标 | 事件结果定价 | 代币对汇率定价 |
| 状态变量 | q_1, q_2, ..., q_n + b | x, y (两种代币余额) |
| 报价公式 | softmax(q_i/b) | y/(x+y) |
| LP 模型 | 内置（seed 承担损失） | 外部 LP 提供流动性 |
| 结算 | 有终止条件（赢/输） | 无终止（持续交易） |
| 多结果 | 天然支持（n 维 softmax） | 需要多池组合 |
| 无常损失 | 不适用（做市损失有上界） | 有（且无上界） |

## 何时不用 LMSR

```
不适合 LMSR 的场景:
  - 高频交易（Gas 太贵）
  - 大额交易（滑点固定，不如订单簿灵活）
  - 已有充足流动性的热门市场（订单簿更高效）

适合 LMSR 的场景:
  - 长尾市场（小众命题）
  - 冷启动阶段
  - 完全链上、无需信任链下组件
  - 教学与原型验证
```

## 自检

1. 为什么 DEX 不用 LMSR？（答：DEX 交易没有终止条件，不需要 softmax 归一化到概率空间）
2. 如果一个预测市场每天有 10 万笔交易，还需要 LMSR 吗？（答：不一定，高流动性市场用订单簿更高效）

# 17.17 LMSR 的直觉理解

上一节（17.16）给了公式推导。本节用数值和图形建立**直觉**——不看公式也能理解 LMSR 在做什么。

## 直觉 1：越多人买越贵

```
初始: q_yes = 0, q_no = 0, b = 1000

  p_YES = e^0 / (e^0 + e^0) = 0.5 / 1.0 = 0.50（50¢）

有人买 500 YES: q_yes = 500, q_no = 0

  p_YES = e^0.5 / (e^0.5 + e^0) = 1.649 / 2.649 = 0.622（62.2¢）

又有人买 500 YES: q_yes = 1000, q_no = 0

  p_YES = e^1.0 / (e^1.0 + e^0) = 2.718 / 3.718 = 0.731（73.1¢）

再买 500: q_yes = 1500, q_no = 0

  p_YES = e^1.5 / (e^1.5 + e^0) = 4.482 / 5.482 = 0.818（81.8¢）

趋势:
  每次买入都推高价格
  但推高的幅度越来越小（因为指数函数）
  → 价格永远在 (0, 1) 之间，不会到 0 也不会到 1
```

### 价格轨迹表

```

───────┼────────┼─────────┼────────┼──────────────────────────
    0  │     0  │  0.500  │ 0.500  │    0
  100  │     0  │  0.525  │ 0.475  │   51.3

（b = 1000，所有数值由 C(q) = b × ln(e^(q/b) + e^0) 计算）
```

## 直觉 2：永远可以交易

```
订单簿模型:
  想买 YES @0.60 → 需要有人卖 YES @0.60
  如果没人卖 → 无法成交

LMSR 模型:
  想买 10 份 YES → 成本 = C(q_yes+10, q_no) - C(q_yes, q_no)
  这是一个纯数学计算 → 不需要对手方
  → 永远有报价

代价:
  「永远可以交易」= 「合约本身承担了对手方风险」
  → 如果所有人都正确预测，合约亏钱
  → 亏损上界 ≈ b × ln(2) ≈ 693（当 b = 1000）
```

## 直觉 3：买卖是对称的

```
买 100 YES:
  cost = C(q_yes + 100, q_no, b) - C(q_yes, q_no, b)
  → 正数 → 用户付钱

卖 100 YES:
  refund = C(q_yes, q_no, b) - C(q_yes - 100, q_no, b)
  → 正数 → 用户收钱

但金额不对称:
  买入时的成本 > 卖出时的退款（如果中间有其他交易）
  → 这就是「价格冲击」的来源

数值例子:
  状态: q_yes = 500, b = 1000
  买 100 YES: cost = C(600,0) - C(500,0) = 57.4 USDC
  此时 q_yes = 600
  立刻卖 100 YES: refund = C(600,0) - C(500,0) = 57.4 USDC
  → 立刻买卖 → 零损失（不含手续费）

  但如果中间有人买了 200 YES:
  卖 100 YES: refund = C(800,0) - C(700,0) = 62.9 USDC
  → 反而赚了（因为价格被推高了）
```

## 直觉 4：b 控制「弹性」

```
小 b (b = 100):
  买 100 YES: p_YES 从 0.50 → 0.73
  → 价格剧烈变化（弹性低、滑点大）

大 b (b = 10000):
  买 100 YES: p_YES 从 0.50 → 0.502
  → 价格几乎不变（弹性高、滑点小）

b 的角色:
  小 b → 市场对小额交易就敏感 → 适合「快速捕捉信息」
  大 b → 需要大额交易才移动价格 → 适合「稳定市场」
  → 但大 b 需要更多资金覆盖最坏损失
```

## 直觉 5：与 CPMM 的本质差异

```
CPMM (x × y = k):
  交易的是「代币 A 和代币 B 的相对价值」
  x, y 是池中实际代币余额
  价格 = y/x → 取决于池中余额比

LMSR:
  交易的是「事件结果的可信度」
  q_yes, q_no 是「做市状态变量」（不是真正的代币余额）
  价格 = softmax(q/b) → 取决于状态和参数

关键差异:
  CPMM 的价格可以从 0 到 ∞（没有上界）
  LMSR 的价格永远在 (0, 1) 且 sum = 1
  → LMSR 天然适合「概率空间」
  → CPMM 天然适合「汇率空间」

  CPMM 需要外部 LP 提供两种代币
  LMSR 只需要单一抵押品 + seed
  → 预测市场不需要「两种代币」
```

## 自检

1. 如果 `p_YES = 0.99`，还能买 YES 吗？成本有多高？（答：可以，但边际成本接近 1，几乎无利可图）
2. 为什么 LMSR 的价格永远不会真正到 0 或 1？（答：softmax 函数的输出域是 (0,1) 开区间，指数函数永远 > 0）

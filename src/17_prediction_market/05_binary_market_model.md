# 17.5 二元市场模型

## 从「一句话命题」到「两枚代币」

二元预测市场的核心逻辑非常简单：

```
命题: "2025 年 1 月 1 日前 BTC 突破 $150K？"
结果空间: { YES, NO }

结算规则:
  如果 BTC 在截止日 >= $150K → YES 赢
  如果 BTC 在截止日 <  $150K → NO 赢

赢家每份获得 1 USDC
输家每份获得 0 USDC
```

## 为什么是「两枚互补代币」

```
Alice 觉得 BTC 会突破 → 她想买 YES
Bob 觉得 BTC 不会突破 → 他想买 NO

如果 YES 的价格是 0.6 USDC，NO 的价格是 0.4 USDC:

  Alice 花 0.6 USDC 买 1 YES
    如果赢 → 获得 1 USDC，净赚 0.4
    如果输 → 获得 0，净亏 0.6

  Bob 花 0.4 USDC 买 1 NO
    如果赢 → 获得 1 USDC，净赚 0.6
    如果输 → 获得 0，净亏 0.4

关键约束: YES 价格 + NO 价格 = 1（否则存在套利）
```

### 套利检查

```
如果 YES = 0.55, NO = 0.35 → 总价 0.90 < 1
  → 套利者可以同时买 1 YES + 1 NO，花 0.90
  → 无论结果如何都能赎回 1.00
  → 无风险利润 0.10

如果 YES = 0.65, NO = 0.45 → 总价 1.10 > 1
  → 套利者可以拆分 1 单位抵押，获得 1 YES + 1 NO
  → 分别以 0.65 和 0.45 卖出，收入 1.10
  → 无风险利润 0.10

结论: 均衡时 P_YES + P_NO = 1
```

## 结果空间的形式化

```
结果空间 Y = {Y, N}，满足:
  - 互斥: 不可能同时 YES 且 NO
  - 完备: 必定有且仅有一个结果为真

Arrow-Debreu 证券:
  YES token = 支付 1 当且仅当 Y 发生
  NO  token = 支付 1 当且仅当 N 发生

这与 Arrow-Debreu 完全市场的基础结构一致
但链上实现需要回答: 谁来判定 Y 还是 N 发生了？
→ 这是 Oracle / Resolution 问题（17.26）
```

## 为什么二元市场最先落地

| 维度 | 二元市场 | 多结果市场 | Scalar 市场 |
|------|---------|-----------|------------|
| 结果定义 | 是/否 | A/B/C/D/... | 连续数值 |
| 合约复杂度 | 低 | 中 | 高 |
| 裁决难度 | 低（通常可自动） | 中 | 高（边界争议） |
| 条件代币 | 2 种 | n 种 | 需要分桶 |
| LMSR 实现 | 二维 softmax | n 维 softmax | n 维 + 桶映射 |

## 「隐含胜率」不等于「真实概率」

```
链上可观测的:
  LMSR 边际价格 p_YES = e^(q_Y/b) / (e^(q_Y/b) + e^(q_N/b))
  → 这是做市公式的数学结果

不可链上验证的:
  "BTC 突破 150K 的真实概率" — 这是一个认识论问题
  → 边际价格受流动性、手续费、参与者偏差影响
  → 本书称之为「隐含胜率」而非「概率」

前端展示建议:
  "YES: 60¢" 或 "隐含胜率 60%"
  而非 "BTC 有 60% 概率突破 150K"
```

## 与本章代码的对应

```
pm.move 中的核心状态:
  Market.q_yes: u64   ← LMSR 做市状态（YES 侧库存）
  Market.q_no: u64    ← LMSR 做市状态（NO 侧库存）

Position 中的用户头寸:
  Position.yes: u64   ← 用户持有的 YES 份额
  Position.no: u64    ← 用户持有的 NO 份额

这两组数字的含义不同:
  q_yes/q_no → 用于 LMSR 定价公式
  Position.yes/no → 用于结算时的赎回
  → 17.22 节展示它们如何在交易函数中协调
```

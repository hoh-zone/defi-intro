# 9.4 CDP 稳定币：机制、锚定套利与风险参数

## CDP 是什么

**CDP（Collateralized Debt Position）**：用户存入链上抵押品，协议**铸造**稳定币作为负债；偿还稳定币后可取回抵押品。与法币抵押不同，**抵押品与债务关系全链可见**，清算与预言机逻辑由合约执行。

## 锚定如何靠套利维持

稳定币市价偏离 $1 时，两类套利（理想化模型）：

**铸造侧（价格 > $1）**：抵押并铸造 → 市场卖出 → 增加供给，压低价格。

**赎回侧（价格 < $1）**：市场买入稳定币 → 偿还债务 → 释放抵押品获利 → 减少供给。

现实中 CDP 系统的「赎回」往往通过**偿还债务 + 提取抵押**完成；是否存在二级市场深度、清算是否及时，决定锚定质量。

## 风险参数体系

| 参数           | 含义                         | 说明                                     |
| -------------- | ---------------------------- | ---------------------------------------- |
| 最低抵押率 MCR | 开仓时抵押价值 / 债务 的下限 | 越高越安全，资本效率越低                 |
| 清算阈值 LR    | 低于该比率可触发清算         | 必须 **低于** MCR，留出缓冲带            |
| 清算罚金       | 清算时从抵押中扣除的比例     | 激励清算人，惩罚借款人                   |
| 债务上限       | 系统总债务上限               | 限制铸币速度、降低系统性风险             |
| 稳定费         | 债务年化成本                 | 覆盖风险与协议收入（实现上常随时间累积） |

关系示意：

```
MCR（如 150%）> LR（如 130%）> 100%
     ↑                ↑
  开仓检查          清算触发
```

## 压力传导示例

假设用户存入价值 $3000 的抵押品，借出 **$2000** 稳定币（名义美元计价），MCR 对应最大可借额度由价格与公式决定。当抵押品价格下跌，抵押率逼近 **LR** 时，清算人可代为偿还债务并获取抵押品（含罚金），保护系统不出现净负债。

若**预言机延迟**或**流动性枯竭**，系统可能在信息或执行层面无法及时清算——这是 **CDP + 预言机 + 市场结构** 的联合风险，而非单一公式可解决。

## 与法币、算法路线的关系

- **法币抵押**：锚定强依赖发行方储备；链上模型薄。
- **CDP**：锚定强依赖**抵押品市场 + 预言机 + 清算激励**；链上模型厚，**本书代码主线**。
- **算法**：锚定依赖规则与经济参与者在压力下的行为；需单独讨论失败模式（见 **9.6**）。

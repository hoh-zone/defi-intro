# 9.3 法币稳定币与 Sui：原生资产、桥与合规边界

## 用户视角：你在 Sui 上持有的「美元稳定币」从哪来

常见来源包括：

1. **原生发行**：发行方在 Sui 上直接部署官方代币包（如 Circle 的 **USDC on Sui**），通过合规渠道铸销。
2. **跨链桥入**：从以太坊等网络桥接封装资产（名称与风险因桥而异）。
3. **包装与合成**：由第三方协议托管他链资产并发行映射代币——须单独审计桥与托管方。

对开发者而言：**包 ID、代币类型 `Coin<...>`、小数位、是否冻结** 必须在集成前逐项核对官方文档。

## 原生发行 vs 桥接资产的风险差异

### 原生发行

发行方直接在 Sui 上部署官方代币包。以 USDC on Sui 为例：

- **包地址由 Circle 控制**：铸销权限在 Circle 手中
- **Coin 类型**：`Coin<0x...::usdc::USDC>`，TypeTag 必须与官方一致
- **小数位**：6 位（1 USDC = 1,000,000 最小单位）
- **冻结能力**：Circle 可以冻结特定地址的 USDC

风险集中在**发行方的信用与合规操作**。

### 桥接资产

通过跨链桥从以太坊等网络桥入的稳定币。

- **名称可能相同但类型不同**：桥接的 USDC 可能叫 `USDC.eth` 或 `axlUSDC`
- **风险叠加**：原链资产风险 + 桥的安全风险 + 目标链合约风险
- **去锚风险**：如果桥出问题，桥接版 USDC 可能大幅脱锚

**开发者注意**：原生 USDC 和桥接 USDC 是**不同的 Coin 类型**，不能互换。在 DEX 池和借贷协议中必须明确使用哪个。

### 风险对比表

| 维度         | 原生发行                 | 桥接资产                       |
| ------------ | ------------------------ | ------------------------------ |
| 合约风险     | 仅发行方合约             | 原链 + 桥 + 目标链三重风险     |
| 脱锚风险     | 低（依赖发行方储备）     | 中高（桥出问题时可能脱锚）     |
| 冻结风险     | 有（发行方可冻结地址）   | 取决于桥的设计                 |
| 审计覆盖     | 发行方审计 + 链审计      | 需要额外审计桥合约             |
| 流动性       | 通常更好                 | 取决于桥的使用量               |

## DeFi 集成时的检查清单

- **官方类型定义**：使用官方发布的 `TypeTag`，勿自行伪造同名代币。
- **价格源**：法币稳定币在美元计价业务中常当作 $1，但极端行情或脱锚时仍应用预言机或聚合价（见第 5 章）。
- **合规与黑名单**：部分稳定币合约支持冻结地址；协议若假设「代币永远可自由流动」，可能与现实不符。

### 集成代码示例

在 Move 中引用特定稳定币类型时：

```move
// 正确：使用完整类型路径
public fun deposit_usdc(
    pool: &mut Pool,
    coin: Coin<0xc0ffee...::usdc::USDC>, // 使用实际包地址
    ctx: &mut TxContext,
) { ... }

// 错误：使用泛型可能接受假币
public fun deposit_stable<T>(
    pool: &mut Pool<T>,
    coin: Coin<T>,
    ctx: &mut TxContext,
) { ... }
// 任何人都可以部署一个也叫 USDC 的假币
```

对于白名单机制：

```move
// 使用动态字段维护允许的稳定币列表
public fun is_allowed_stable(pool: &Pool, type_name: String): bool {
    dynamic_field::exists_(&pool.id, type_name)
}
```

## 与本书 CDP 教程的关系

- **法币稳定币**常作为 CDP 系统中的**债务计价单位**或**多抵押品之一**（例如抵押 ETH 借 USDC）。此时协议同时承担：
    - CDP 侧：清算、预言机、抵押率；
    - 稳定币侧：该法币资产自身的对手方与合规风险。

当 CDP 使用法币稳定币作为抵押品时，协议需要额外考虑：
- 稳定币的冻结风险：如果用户抵押的 USDC 被冻结，协议如何处理？
- 脱锚风险：USDC 跌到 $0.95 时，抵押品估值是否需要调整？
- 桥接 vs 原生：协议是否同时接受两种来源的同名稳定币？

## 小结

法币抵押路线在链上的「实现」往往很薄：**薄在代码，厚在信任与合规**。理解这一点有助于与 **9.4 以后** 的 CDP（全链可验证抵押）形成正确对比。

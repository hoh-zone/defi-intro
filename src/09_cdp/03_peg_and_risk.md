# 9.3 锚定机制与风险参数体系

## 稳定币如何维持锚定

稳定币的价格为什么能维持在 $1 附近？靠的是两套激励：

### 铸造激励（Hard Peg 上限）

当稳定币价格 > $1 时，套利者可以存入抵押品、铸造稳定币、在市场卖出获利。这增加了供给，压低价格。

```
稳定币价格 = $1.05
→ 套利者存入 $1.5 抵押品，铸造 $1 稳定币（成本 $1）
→ 在市场卖出获得 $1.05
→ 利润 $0.05
→ 供给增加，价格回落到 $1
```

### 赎回激励（Hard Peg 下限）

当稳定币价格 < $1 时，套利者可以在市场买入稳定币，用它在协议中赎回抵押品获利。这减少了供给，推高价格。

```
稳定币价格 = $0.95
→ 套利者用 $0.95 买入 1 稳定币
→ 在协议中偿还 1 稳定币，取回 $1.5 抵押品
→ 卖出抵押品获得 $1.5
→ 利润 $0.55
→ 供给减少，价格回升到 $1
```

这两个激励形成了一个套利通道。只要套利有利可图，锚定就能维持。当套利不再有利（如抵押品价值暴跌），锚定就会打破。

## 风险参数体系

| 参数 | 含义 | 典型值 | 调整逻辑 |
|------|------|--------|----------|
| 最低抵押率 (MCR) | 开仓时必须满足的最低比例 | 150% | 越高越安全，但资本效率越低 |
| 清算阈值 (LR) | 触发清算的抵押率 | 130% | 必须 < MCR |
| 清算罚金 | 清算时从抵押品中扣除的比例 | 5-15% | 越高清算者越积极，借款人损失越大 |
| 稳定费 | 年化借贷利率 | 1-5% | 用于覆盖运营成本和坏账 |
| 债务上限 | 系统总债务的最大值 | 按需设定 | 防止供给过快增长 |
| 预言机更新频率 | 价格更新的最小间隔 | 1-5 分钟 | 太快浪费 Gas，太慢有风险 |

### 参数之间的关系

```
最低抵押率 (150%) > 清算阈值 (130%) > 100%
         ↑                        ↑
    开仓时检查              清算时检查
```

差值空间 = 150% - 130% = 20%，这是抵押品价格可以下跌的空间。在这 20% 的空间内，仓位不会被清算，但也不能继续借出。

## 压力传导示例

初始状态：
- 用户存入 100 SUI（$2/枚，价值 $200）
- 借出 100 USDs
- 抵押率 = 200%

| SUI 价格 | 抵押品价值 | 抵押率 | 状态 |
|----------|-----------|--------|------|
| $2.0 | $200 | 200% | 安全 |
| $1.5 | $150 | 150% | 安全（刚好在最低线） |
| $1.35 | $135 | 135% | 危险（接近清算） |
| $1.29 | $129 | 129% | **可被清算** |
| $1.0 | $100 | 100% | 严重资不抵债 |

如果 SUI 在 5 分钟内从 $1.5 跌到 $1.0，而预言机更新不及时，协议就会在 $1.0 时仍认为仓位是安全的。坏账就是这样产生的。

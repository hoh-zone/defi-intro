# 第 15 章 Launchpad 与代币发行

## Launchpad 不只是一个付款页面

Launchpad 的核心是**按规则分配代币**。它需要解决：

- 谁有资格参与？（白名单）
- 怎么分配才公平？（配售机制）
- 怎么防止 Bot 和 Sybil 攻击？
- 代币如何逐步释放？（归属机制）

这些问题的答案是**状态机设计**——Launchpad 本质上是一个有明确状态转换的有限状态机。


## 本章目标

- 理解代币发行的状态机、白名单、配售、定价和领取流程。
- 比较荷兰式拍卖、英式拍卖、固定价和按比例分配。
- 掌握 Vesting、AntiBot、退款和二级市场衔接的设计要点。
- 能评估 Launchpad 的公平性、合规边界和市场风险。

## 先修知识

- 理解 Coin 铸销、时间戳、Capability 和状态机。
- 了解一级发行和二级流动性的差异。

## 本章小结

Launchpad 的核心不是“卖币”，而是把资格、价格、时间、额度和领取权写成可验证状态机。发行机制的公平性和风险往往来自规则边界，而不是单个函数的复杂度。

## 练习题

1. 画出白名单、认购、结算、领取四阶段状态机。
2. 比较荷兰式拍卖和固定价销售对抢跑的影响。
3. 设计一个防重复领取的 ClaimRecord。
4. 说明 Vesting 规则如何影响二级市场抛压。

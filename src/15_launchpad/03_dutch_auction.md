# 15.3 荷兰式拍卖：从高到低的价格发现

## 固定价格的困境

固定价格发售有一个根本问题：**价格谁来定？**

- 定高了 → 卖不完，项目方融资不足
- 定低了 → 瞬间抢光，项目方吃亏，用户也觉得不公平

荷兰式拍卖（Dutch Auction）让市场自己发现价格。

## 核心原理

价格从一个较高的起始价开始，随时间线性下降。当某个买家认为价格合适时，他出价购买——此时拍卖结束，该价格就是成交价。

```
价格
  │
10 ┤●
   │  ●
   │    ●
   │      ●
 6 ┤─────────── ● ← Alice 买入，成交价 = 6
   │              ●
   │                ●
 2 ┤────────────────────●
   └────────────────────── 时间
   0min              60min
```

## 数学模型

### 线性衰减公式

给定参数：

- `start_price`：起始价格（最高价）
- `end_price`：最低价格（地板价）
- `start_time`：拍卖开始时间戳
- `duration`：拍卖持续时间（毫秒）

在任意时刻 `now`，当前价格为：

```
elapsed = now - start_time
current_price = start_price - (start_price - end_price) * elapsed / duration
```

约束：

- `elapsed < 0` → 拍卖尚未开始
- `elapsed >= duration` → 价格已降至地板价 `end_price`
- `start_price > end_price`（否则不是衰减）

### 数值验证

参数：`start_price = 10 SUI`，`end_price = 2 SUI`，`duration = 3600s`（1 小时）

| 时间点 | elapsed (s) | 计算过程             | current_price     |
| ------ | ----------- | -------------------- | ----------------- |
| 0 min  | 0           | 10 - 8 × 0 / 3600    | 10.0 SUI          |
| 15 min | 900         | 10 - 8 × 900 / 3600  | 8.0 SUI           |
| 30 min | 1800        | 10 - 8 × 1800 / 3600 | 6.0 SUI           |
| 45 min | 2700        | 10 - 8 × 2700 / 3600 | 4.0 SUI           |
| 60 min | 3600        | 10 - 8 × 3600 / 3600 | 2.0 SUI（地板价） |

验证：价格从 10 线性降到 2，每 15 分钟下降 2 SUI。

### 购买逻辑

买家支付 `payment_amount` 的 SUI，获得的代币数量：

```
token_amount = payment_amount / current_price
```

所有买家获得相同的代币，但支付的价格可能不同（因为购买时间不同）。

## 与固定价格发售对比

| 维度       | 固定价格           | 荷兰式拍卖               |
| ---------- | ------------------ | ------------------------ |
| 定价方式   | 项目方指定         | 市场发现                 |
| 价格确定性 | 高（提前知道）     | 低（到那一刻才知道）     |
| 公平性     | 取决于定价         | 时间公平：先到者付更高价 |
| Bot 防御   | 差                 | 好（需要在价格窗口决策） |
| 卖完速度   | 取决于定价         | 一定能卖出（到地板价时） |
| 适用场景   | 价格已知的社区发售 | 价格不确定的新项目       |

## 荷兰式拍卖的三重优势

### 1. 价格发现

不需要项目方猜测市场愿意付多少钱。市场通过出价时机来表达对价格的判断。

### 2. 反 Bot

在固定价格发售中，Bot 可以在开售的瞬间抢购。在荷兰式拍卖中：

- 早期价格高，Bot 不愿买
- 价格降到合理区间时，真人用户和 Bot 竞争
- 但 Bot 的速度优势被削弱——因为它需要在"价格是否合理"上做决策

### 3. 保证成交

只要地板价设得合理，代币一定能在拍卖期内卖出。不存在"卖不完"的风险。

## 时间压力与策略

荷兰式拍卖制造了时间压力：

- **等着降价**：可能被别人抢先
- **现在就买**：多付了钱

这种博弈让每个买家在"价格"和"确定性"之间权衡，最终成交价反映了市场的集体判断。

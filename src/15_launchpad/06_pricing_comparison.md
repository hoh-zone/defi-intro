# 15.6 定价机制对比与选择

## 三种定价机制一览

Launchpad 的核心决策是：**怎么给代币定价？**

目前我们讨论了三种机制：

### 固定价格（Fixed Price）

```
价格 = 项目方指定的常量

Alice 买 100 token → 支付 100 × price
Bob   买 200 token → 支付 200 × price
```

### 荷兰式拍卖（Dutch Auction）

```
价格 = f(时间)，从高到低线性衰减

Alice 在 t=15min 买 → 价格 = 8 SUI/token
Bob   在 t=30min 买 → 价格 = 6 SUI/token
```

### 英式拍卖（English Auction）

```
价格 = 出价者博弈的结果，从低到高

Alice 出价 2 SUI → Bob 出价 3 SUI → Carol 出价 5 SUI
赢家：Carol，成交价：5 SUI
```

## 全面对比表

| 维度           | 固定价格             | 荷兰式拍卖               | 英式拍卖               |
| -------------- | -------------------- | ------------------------ | ---------------------- |
| **定价者**     | 项目方               | 市场（时间）             | 市场（竞价）           |
| **价格确定性** | 确定                 | 不确定                   | 不确定                 |
| **价格方向**   | 恒定                 | ↓ 下降                   | ↑ 上升                 |
| **卖完保证**   | 不确定（取决于定价） | 确定（到地板价一定卖出） | 确定（只要有人出价）   |
| **资金效率**   | 高（支付即完成）     | 高（支付即完成）         | 低（多方资金锁定）     |
| **Gas 成本**   | 低                   | 低                       | 高（多轮竞价）         |
| **Bot 防御**   | 差                   | 好                       | 一般                   |
| **信息公开**   | 完全公开             | 当前价格公开             | 所有出价公开           |
| **Sybil 防御** | 弱（需要额外白名单） | 中（价格决策门槛）       | 弱（可创建多账户出价） |
| **适用规模**   | 任意                 | 适合大量代币             | 适合少量/稀缺资产      |
| **实现复杂度** | 低                   | 中                       | 高                     |

## 定价博弈分析

### 固定价格：定价者的困境

```
                  真实价值线
                      │
    定高了 → ─ ─ ─ ─ ─┼─ ─ ─ ─ ─  项目方吃亏
                      │
    定准了 → ─ ─ ─ ─ ─●─ ─ ─ ─ ─  理想情况
                      │
    定低了 → ─ ─ ─ ─ ─┼─ ─ ─ ─ ─  瞬间抢光，社区不满
                      │
```

固定价格把定价风险完全放在项目方身上。

### 荷兰式拍卖：时间的博弈

```
买家的两难：

    早买 → 价格高，但确保买到
    晚买 → 价格低，但可能被抢走

    这种博弈导致成交价自然收敛到市场共识。
```

### 英式拍卖：价格的博弈

```
买家的两难：

    继续加价 → 可能赢得拍品，但付更多
    放弃 → 确定失去拍品，但省钱

    最终赢家是"愿意付最多的人"，即代币在其手中价值最高。
```

## 选择决策树

```
代币发售量？
├── 大量代币（> 1M）
│   ├── 价格是否已知？
│   │   ├── 是 → 固定价格发售
│   │   └── 否 → 荷兰式拍卖
│   └── 需要 Bot 防御？
│       ├── 是 → 荷兰式拍卖
│       └── 否 → 固定价格
├── 少量/稀缺资产
│   └── 英式拍卖
└── 不确定
    └── 荷兰式拍卖（最安全的默认选择）
```

## Sui 生态中的实践

Sui 上的 Launchpad 项目通常支持多种定价模式：

| 项目               | 固定价格 | 荷兰式 | 英式 | 特色           |
| ------------------ | -------- | ------ | ---- | -------------- |
| SuiPad             | ✓        | ✓      | -    | 白名单 + FCFS  |
| Aftermath          | ✓        | -      | -    | 与生态积分结合 |
| 其他 Sui Launchpad | ✓        | 部分   | 部分 | 多模式支持     |

**趋势**：越来越多的 Launchpad 支持多种定价模式，让项目方根据自身情况选择。一个设计良好的 Launchpad 应该把定价策略作为可配置参数，而不是写死在合约中。

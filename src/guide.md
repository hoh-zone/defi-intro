# 导读：谁适合读这本书与阅读路径

## 目标读者

本书假设你具备以下基础：

- 至少一种编程语言的经验（不限于 Move）
- 对区块链基本概念有了解（交易、区块、共识）
- 对 DeFi 有模糊认知但未系统学习

我们不假设你有 Sui 或 Move 的开发经验。第二篇会覆盖必要的语言基础。如果你完全没接触过智能合约，建议先完成 Sui 官方教程的前三个章节。

### 三类典型读者

**开发者**：你想在 Sui 上构建 DeFi 协议或贡献现有协议。你是本书的首要读者。建议逐章阅读，动手复现每个代码片段。

**产品研究者**：你需要理解 DeFi 协议的运作逻辑以评估可行性或风险敞口。你可以跳过第二篇的 Move 语言细节，重点阅读机制讲解和风险分析部分。

**教学者**：你需要结构化的材料来教授 DeFi 课程。本书的六篇结构可以直接映射为教学大纲：前三篇是基础模块，后三篇是专题模块。

## 三条阅读路径

### 路径 A：完整路线（约 40 小时）

适合有充足时间且希望系统掌握的读者。按章节顺序阅读，每章完成后动手编写或修改代码。

| 阶段 | 内容 | 建议时长 |
|------|------|----------|
| 基础 | 第一篇（第1-3章） | 8 小时 |
| 价格层 | 第二篇（第4-6章） | 8 小时 |
| 信用层 | 第三篇（第7-9章） | 8 小时 |
| 收益层 | 第四篇（第10-16章） | 10 小时 |
| 安全层 | 第五篇（第17-19章） | 6 小时 |

### 路径 B：构建者路线（约 20 小时）

适合已有 DeFi 认知、需要快速上手 Sui 开发的读者：

1. 第2章：Sui 对象模型与 Move 精要（必读）
2. 第4章：DEX（理解流动性池的对象设计）
3. 第7章：借贷（理解利率和清算机制）
4. 第18章：协议工程化实践（必读）
5. 附录B：核心公式

### 路径 C：审计者路线（约 15 小时）

适合关注安全审计的读者：

1. 第2章：Move 资源语义与安全边界
2. 第3章：风险分类体系与五问法
3. 第17章：攻击模式全览
4. 第19章：审计准备与权限矩阵
5. 附录D：延展阅读中的审计报告

## 统一分析框架：五问法

本书每一章分析具体协议时，都使用同一个分析框架：

1. **资产从哪来？** — 谁向协议提供了什么资产？以什么形式？有无锁定条件？
2. **资产到哪去？** — 资产在协议内部如何流转？哪些路径可以提取？哪些被锁定？
3. **谁可以动？** — 哪些角色拥有什么权限？权限如何表达和约束？
4. **价格从哪来？** — 价格数据来自哪里？更新频率？容错机制？
5. **失败会怎样？** — 极端情况下协议的行为？有无熔断机制？损失由谁承担？

这五个问题构成了评估任何 DeFi 协议的最小充分集。

## 术语约定

| 写法 | 含义 |
|------|------|
| **英文术语** | 首次出现时加粗并附中文解释，后续直接使用英文 |
| `代码字体` | 变量名、函数名、命令行指令 |

全书保留英文不翻译的术语：AMM, DEX, CDP, LP, TVL, APY, APR, Oracle, Liquidation, Slippage, Flash Loan, PTB, Object, Ability, Capability。

所有金额单位默认为最小精度（如 SUI 的 MIST，1 SUI = 10^9 MIST），除非特别标注。

## 环境准备

阅读本书时建议同步准备以下环境：

```bash
# 安装 Sui CLI
cargo install --locked --git https://github.com/MystenLabs/sui.git --branch mainnet sui

# 验证安装
sui --version

# 创建工作目录
mkdir sui-defi-book && cd sui-defi-book
sui move new examples
```

后续章节的代码示例都可以在 `examples` 包中创建和测试。

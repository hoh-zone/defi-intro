# 4.28 Hybrid DEX 设计

AMM 和 Orderbook 各有优劣。Hybrid DEX 将两者结合，试图获得两者的优势。FlowX Finance 是 Sui 上 Hybrid DEX 的代表。

## 为什么要混合

```
AMM 的优势:
  ✅ 低门槛（任何人可以提供流动性）
  ✅ 即时成交（永远有流动性）
  ✅ 适合长尾资产

AMM 的劣势:
  ❌ 无常损失
  ❌ 大额交易滑点大
  ❌ 无法做限价单

Orderbook 的优势:
  ✅ 精确定价
  ✅ 限价单
  ✅ 无 IL 做市

Orderbook 的劣势:
  ❌ 门槛高（需要做市策略）
  ❌ 长尾资产流动性差
  ❌ 如果没有 Maker 就无法交易

Hybrid: 结合两者 → 在所有场景下都有好的交易体验
```

## 流动性共享

### AMM 价格作为参考

```
AMM 池的价格 = 市场均衡价格的近似
Orderbook 的中间价应该接近 AMM 价格

当两者价格不一致:
  → 套利者在两者之间搬砖
  → 价格自动对齐
```

### Orderbook 挂单增强 AMM 流动性

```
场景: 用户提交大额市价卖单

纯 AMM:
  → 全部通过 AMM 池执行
  → 高滑点

Hybrid:
  1. 先与 Orderbook 的买盘撮合（限价买单）
  2. 剩余部分通过 AMM 池执行
  3. 总滑点更低

例:
  卖出 10,000 SUI
  Orderbook 买盘: 3,000 SUI @ $2.00 (零滑点)
  AMM 池: 7,000 SUI → 获得 $13,500 (滑点 3.6%)
  平均滑点: 1.2%（比纯 AMM 的 2.5% 更好）
```

## 路由逻辑

### 交易路由决策

```
新交易到达
    ↓
分析交易类型:
    ↓
┌─────────┬──────────┬───────────┐
│ 市价小单 │ 市价大单  │ 限价单     │
│ (< $1K)  │ (> $1K)   │           │
├─────────┼──────────┼───────────┤
│ 直接 AMM │ 路由优化  │ 写入 OB   │
│          │ AMM + OB  │ 等待撮合  │
└─────────┴──────────┴───────────┘
```

### 路由优化算法

```
目标: 最大化用户输出（最小化总成本）

方法:
  1. 计算 AMM 单独执行的输出
  2. 计算 Orderbook 单独执行的输出
  3. 计算混合执行的输出（部分走 AMM，部分走 OB）
  4. 选择最优方案

混合执行的最优拆分:
  在 AMM 和 Orderbook 之间找到边际价格相等的点
  → 此时总输出最大
```

## 实现挑战

### 1. 状态一致性

```
问题: AMM 和 Orderbook 的状态需要保持一致

场景:
  AMM 价格: $2.00
  Orderbook 最低卖价: $1.98（应该已经被套利吃掉）

解决:
  → 套利者自动修正（经济激励）
  → 或在路由时检测并修正（代码逻辑）
```

### 2. 原子性

```
问题: 一笔交易可能涉及 AMM 和 Orderbook 两个组件

Sui PTB 的解决方案:
  PTB:
    1. 在 Orderbook 撮合部分交易
    2. 在 AMM 池执行剩余部分
    3. 合并输出返回用户

  原子性: 如果步骤 2 失败，步骤 1 自动回滚
```

### 3. Gas 成本

```
Hybrid DEX 的 Gas 分析:

纯 AMM Swap: ~5M Gas
纯 Orderbook 撮合: ~8M Gas
Hybrid 路由: ~12M Gas（两个组件都涉及）

Sui 的低 Gas 费使得 Hybrid 方案在经济上可行
（在 Ethereum 上，额外的 Gas 成本可能抵消路由优化的收益）
```

## 其他 Hybrid 设计模式

### 模式一：AMM + 批量拍卖

```
定期（如每 5 秒）将所有 AMM 交易和订单簿挂单一起撮合
→ 批量撮合减少 Gas
→ 更公平的价格发现（减少 MEV）

代表：CoW Protocol（以太坊）
```

### 模式二：AMM + 意图（Intent）

```
用户提交"意图"而非具体交易:
  "我想把 1000 SUI 换成至少 1950 USDC"

求解者（Solver）竞争提供最优执行:
  → 可能走 AMM、Orderbook、或其他路径
  → 用户获得最佳价格

代表：UniswapX（以太坊）
```

### 模式三：AMM 内嵌限价单

```
在 CLMM 中，LP 的仓位在价格到达边界时
自动变成单一代币 → 类似于限价单被执行

例: LP 在 $1.90-$2.10 区间提供流动性
  → 价格跌到 $1.90 时，LP 全部变成 A 代币
  → 等价于在 $1.90 挂了一个买单

CLMM 的 "TWAMM" 扩展:
  → 自动将大额订单拆分为小额度，逐步执行
  → 减少 MEV 和价格冲击
```

## Hybrid DEX 的发展趋势

```
趋势一: 路由越来越智能
  → 不仅在 AMM 和 OB 之间选择
  → 还考虑多个 DEX、多个链

趋势二: 意图驱动的交易
  → 用户只需表达意愿
  → 系统自动找到最优路径

趋势三: MEV 保护
  → 通过批量拍卖或加密内存池
  → 减少三明治攻击

Sui 的优势:
  PTB 使得 Hybrid 路由在 Gas 上可行
  对象模型使得状态管理更灵活
  低延迟使得实时路由决策更快
```

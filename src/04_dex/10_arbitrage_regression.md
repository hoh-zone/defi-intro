# 4.10 套利与价格回归

DEX 的价格如何与外部市场保持一致？答案是**套利**（Arbitrage）。套利者是 DEX 生态中不可或缺的角色。

## 为什么需要套利

AMM 的价格完全由池中储备比例决定。没有外部力量，价格永远不会自动调整到"正确"水平。

```
问题场景：
  外部市场 SUI = $3.00
  DEX 池中 SUI = $2.00（被低估）

如果不套利:
  → DEX 价格永远停在 $2.00
  → 所有依赖 DEX 价格的协议都使用错误价格
  → 系统性风险

如果套利:
  → 套利者低价买入 SUI → 池中 SUI 减少
  → SUI 在池中价格上升 → 趋向 $3.00
  → 所有协议使用正确价格
```

## 套利的数学

### 何时套利有利可图

```
外部价格: P_market
池中价格: P_pool = y/x

如果 P_pool < P_market:
  → 池中 A 被低估
  → 套利者：在池中用 B 买 A → 在外部卖 A 获利
  → 利润 = A在外部的价值 - A在池中的成本

如果 P_pool > P_market:
  → 池中 A 被高估
  → 套利者：在外部买 A → 在池中卖 A 获得 B → 获利
```

### 最优套利量

不是套利越多越好。大量套利本身会改变池中价格。

```
最优套利量 Δx 使得:
  池中价格被推到 = 外部价格

即: (y - Δy) / (x + Δx) = P_market

其中 Δy = Δx × y / (x + Δx)（AMM 公式）

解这个方程可以得到最优 Δx。
```

### 数值示例

```
初始状态:
  x(SUI) = 1000, y(USDC) = 2000
  池中价格 = 2.0 USDC/SUI
  外部价格 = 3.0 USDC/SUI

SUI 在池中被低估！套利者买入 SUI。

投入 Δx USDC → 获得 Δy SUI:
  Δy = Δx × 1000 / (2000 + Δx)

套利后池中价格 = (2000 + Δx) / (1000 - Δy)
  = 外部价格 3.0 时达到平衡

近似计算：
  需要将 y/x 从 2.0 调整到 3.0
  解: (2000 + Δx) / (1000 - Δy) = 3

通过迭代或数值方法求解，大约需要投入 ~789 USDC
套利者获得 ~286 SUI，在外部市场值 $857
利润 = $857 - $789 = $68
```

## 套利的速度竞争

套利机会存在的时间窗口非常短。多个套利者竞争同一个机会：

```
套利竞争:
  时间 T: 价格偏差出现
  T + 100ms: 第一个套利 bot 检测到
  T + 200ms: 多个 bot 开始竞争
  T + 400ms: 第一个交易被打包（Sui 出块时间）
  T + 500ms: 价格已修正，套利机会消失
```

### Gas 竞争

在 Gas 费更高的链上（如 Ethereum），套利者需要提高 Gas 价格来竞争：

```
套利利润 = 价差收益 - Gas 费用 - 滑点

如果 Gas 费用 > 价差收益 → 没有人套利 → 价格偏差持续
→ 对依赖 DEX 价格的协议是灾难
```

Sui 的低 Gas 费使得小额套利也有利可图，这意味着价格偏差会被更快修正。

## Sui PTB 的套利优势

Sui 的 PTB 使得套利可以完全原子化：

```
传统链上套利（多笔交易）:
  TX1: 从 DEX 买入 SUI（价格 $2）
  TX2: 在外部卖出 SUI（价格 $3）
  风险: TX1 成功但 TX2 失败 → 持有 SUI 敞口

Sui PTB 套利（原子操作）:
  PTB:
    1. 闪电贷借入 USDC
    2. 在 DEX 用 USDC 买 SUI
    3. 在另一个 DEX 卖 SUI 获得 USDC
    4. 归还闪电贷
    5. 保留利润
  原子性: 要么全部成功，要么全部回滚 → 零风险套利
```

## 套利对 LP 的影响

套利对 LP 的影响是双面的：

```
正面影响:
  ✅ 价格回归 → 减少无常损失
  ✅ 交易量增加 → LP 获得更多手续费
  ✅ 池的定价更准确 → 更可信的价格来源

负面影响:
  ⚠️ 大额套利 → 高滑点 → LP 承受价格冲击
  ⚠️ MEV → 三明治攻击 → LP 和普通交易者受损
  ⚠️ 如果套利来自内部信息 → LP 处于信息劣势
```

## 三明治攻击

三明治攻击是套利的恶意变种：

```
正常套利:
  发现价差 → 交易获利 → 价格回归 → 所有参与者受益

三明治攻击:
  1. 攻击者在 Mempool 中发现受害者的待处理交易
  2. 攻击者在受害者交易前买入（推高价格）
  3. 受害者的交易以更差的价格执行
  4. 攻击者在受害者交易后卖出（获利）

受害者: 以更高的价格买入
攻击者: 利润 = 价格差 × 数量
```

> 三明治攻击的详细分析和防御策略见第 14 章。

## 套利总结

```
套利的角色:
  1. 价格锚定: 将 DEX 价格拉回市场均衡
  2. 流动性激励: 为 LP 提供交易量和手续费
  3. 效率信号: 套利利润高 = 价格效率低

对 DEX 设计者的启示:
  1. 池越大，套利利润越小，价格越稳定
  2. 低 Gas 费（如 Sui）促进更频繁的小额套利
  3. PTB 使得无风险套利成为可能
  4. 需要防御 MEV（三明治攻击）
```

# 4.8 恒定乘积 AMM 数学

4.6 节的最小 Swap 实现有一个致命问题：每次交易后池的总价值会流失。恒定乘积公式 x·y=k 解决了这个问题。

## 从第一性原理推导

### 约束条件

一个"公平"的 Swap 应该满足：

1. **交易前后池的价值不变**（LP 不应该因交易受损）
2. **价格随供需变化**（买入越多，价格越高）

### 推导过程

设池中有代币 A（数量 x）和代币 B（数量 y）。用户投入 Δx 个 A，获得 Δy 个 B。

约束：交易后的乘积等于交易前：

```
交易前: k = x × y
交易后: k' = (x + Δx) × (y - Δy)

令 k = k':
  (x + Δx) × (y - Δy) = x × y

展开:
  xy + Δx·y - x·Δy - Δx·Δy = xy

简化:
  Δx·y - x·Δy - Δx·Δy = 0
  Δx·y = Δy·(x + Δx)

解出 Δy:
  Δy = Δx × y / (x + Δx)
```

这就是恒定乘积 AMM 的核心公式。

### 验证

```
初始状态: x=1000, y=2000, k=2,000,000
用户投入 Δx=100 A

Δy = 100 × 2000 / (1000 + 100) = 200,000 / 1100 = 181.81...

交易后: x=1100, y=1818.18
k' = 1100 × 1818.18 = 1,999,998 ≈ 2,000,000 ✅

（差异来自浮点截断，整数运算中 k 完全守恒）
```

## 含手续费公式

实际交易中，输入金额的一部分作为手续费被扣除：

```
不含手续费:
  Δy = Δx × y / (x + Δx)

含手续费（fee_bps 基点）:
  Δx_with_fee = Δx × (10000 - fee_bps) / 10000
  Δy = Δx_with_fee × y / (x × 10000 + Δx_with_fee)

为什么分母用 x × 10000 + Δx_with_fee 而不是 (x + Δx_with_fee/10000)?
  → 为了避免两次除法带来的精度损失
```

### 手续费使 k 增长

```
交易前: k = x × y
交易后（含手续费）:
  输入 Δx，手续费 = Δx × fee_bps / 10000
  实际入池 = Δx（全部，手续费从输出中体现）

  k' = (x + Δx) × (y - Δy)
  由于 Δy < Δx × y / (x + Δx)（因为手续费减少了有效输入）
  所以 k' > k

结论：每次交易，k 都会增长！这对手续费收入的 LP 来说是好事。
```

## 价格与储备的关系

### 即时价格

```
池中即时价格（每单位 A 值多少 B）:
  P = y / x

例:
  x=1000 SUI, y=2000 USDC
  P = 2000/1000 = 2 USDC/SUI
```

### 价格如何随交易变化

```
初始: x=1000, y=2000, P=2.0
买入 100 SUI:
  Δy = 100×2000/1100 = 181.82
  新状态: x=1100, y=1818.18
  新价格: P = 1818.18/1100 = 1.653 USDC/SUI

价格从 2.0 下降到 1.653 → 买入使被买代币的价格上升（用更少 B 换更多 A）

等一下，这不对。让我重新理解：

Swap A→B: 用户给 A，获得 B
  → 池中 A 增多，B 减少
  → A 相对 B 贬值（A 更多了）
  → 每单位 A 值更少的 B

这是正确的：买入 B（给 A）使 A 贬值、B 升值
```

## k 的经济学含义

```
k 的意义：
  k = x × y 代表池的"流动性深度"

k 越大 → 同样金额的交易，价格冲击越小
k 越小 → 同样金额的交易，价格冲击越大

例：
  Pool A: x=1000, y=2000, k=2,000,000
  Pool B: x=100, y=200, k=20,000

同样投入 10 A:
  Pool A: Δy = 10×2000/1010 = 19.80, 价格冲击 = 1%
  Pool B: Δy = 10×200/110 = 18.18, 价格冲击 = 9.1%

k 差 100 倍 → 价格冲击差约 10 倍
```

## k 增长的来源

| 来源       | 机制           | 对 LP 的影响           |
| ---------- | -------------- | ---------------------- |
| 交易手续费 | 每笔交易增加 k | LP 份额增值            |
| 添加流动性 | LP 存入新资金  | k 增长但 shares 也增加 |
| 协议费     | 从手续费中截留 | LP 增值速度略降        |

## 数学性质总结

```
1. k 守恒（不含手续费）→ 保证 LP 不因交易受损
2. k 增长（含手续费）→ LP 因交易获得收入
3. 价格 = y/x → 价格由储备比例决定
4. 深度 = sqrt(k) → k 越大，价格越稳定
5. 凸性 → 买入越多，边际价格越差（自然抑制大额交易）
```

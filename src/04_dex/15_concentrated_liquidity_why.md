# 4.15 为什么需要集中流动性

CPMM 的流动性均匀分布在 (0, +inf) 的全价格区间，但大多数交易只发生在极窄的价格范围内。集中流动性（Concentrated Liquidity）让 LP 把资金集中在期望的价格区间，大幅提升资本效率。

## CPMM 的资本效率问题

CPMM 的流动性按恒定乘积曲线 x \* y = k 均匀分布在整个价格轴上，但实际交易高度集中在当前价格附近：

```
CPMM 流动性分布:
价格轴: 0 ──────────── P_cur ──────────── +inf
流动性:  ████████████████████████████████████████
         全区间均匀分布

实际交易分布:
价格轴:  0 ── Rare ── P_cur(95%交易) ── Rare ── +inf
交易量:  ░░░░░░░░░░ ██████████████████ ░░░░░░░░░░
```

### 数值示例：80% 的流动性从未使用

```
SUI/USDC 池：P = $2.00, 24h 交易区间: $1.90 - $2.10, TVL = $1,000,000

  区间           | 流动性量    | 24h 利用率
  ───────────────|────────────|──────────
  $0.00 - $1.80  | $450,000   |  0.1%
  $1.80 - $1.90  | $100,000   |  2%
  $1.90 - $2.10  | $300,000   |  95%   ← 核心区间
  $2.10 - $2.20  | $100,000   |  3%
  $2.20 - +inf   | $50,000    |  0.1%

  核心 TVL: $300,000 (30%)    浪费的 TVL: $700,000 (70%)

稳定币对更极端：USDC/USDT 池中 95%+ 的流动性永远闲置。
```

## CLMM 解决方案

CLMM（Concentrated Liquidity Market Maker）允许 LP 选择价格区间 [P_lower, P_upper]，只在该区间提供流动性：

```
CPMM:  LP 提供 $1000 → 均匀分布在 $0 到 +inf
CLMM:  LP 提供 $1000 → 集中在 $1.80 - $2.20
       当前价附近流动性密度提升 ~100 倍
```

### 资本效率数学表达

```
效率倍数 ≈ sqrt(P_cur / range_width) × sqrt(P_cur / range_width)

ETH/USDC 池 (当前 $2000, 区间 $1800-$2200):
  效率倍数 ≈ sqrt(2000/400)^2 ≈ 100 倍

USDC/USDT 池 (区间 $0.999-$1.001):
  效率倍数 ≈ sqrt(1/0.002)^2 ≈ 250000 倍
```

## 集中流动性的代价

```
1. 需要主动管理
   价格移出区间 → 停止赚取手续费 → 需要调整

2. 无常损失集中
   窄区间 ($1.90-$2.10) 内 IL 远大于全区间

3. 多 Position 管理
   每个 Position 是独立 NFT，需分别管理
```

CLMM 通过让 LP 自定义价格区间，实现了数量级的资本效率提升。下一节深入 CLMM 的核心机制——Tick 与价格区间。

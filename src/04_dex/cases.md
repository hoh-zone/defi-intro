# 4.5 Sui 生态 DEX 实例分析

## 三个代表项目

| 维度 | Cetus | Kriya | DeepBook |
|------|-------|-------|----------|
| 类型 | 集中流动性（CLMM） | 多工具（AMM + CLMM + 稳定币互换） | 链上订单簿（CLOB） |
| 核心优势 | 资金效率高 | 一站式交易 | 价格精度高 |
| 适合场景 | 主流交易对的LP | 多策略交易者 | 大额交易、做市商 |
| 价格精度 | 区间内高，区间外无 | 取决于使用的池类型 | 精确到挂单价格 |
| 门槛 | 需要理解集中流动性 | 低 | 需要理解订单簿 |

## Cetus（CLMM）

Cetus 是 Sui 上最大的集中流动性 DEX。它的核心是 CLMM（Concentrated Liquidity Market Maker），类似于 Uniswap V3。

特点：
- LP 可以选择任意价格区间提供流动性
- 支持多费率档位（0.01%、0.05%、0.25%、1%）
- 集成了闪电聚合（Flash Aggregator），跨池路由最优路径
- 支持借贷协议的杠杆做市

## Kriya（多工具）

Kriya 的特点是提供多种交易工具：
- 标准 AMM 池
- 集中流动性池
- 稳定币互换（Stableswap，类似 Curve）
- 限价单功能

多工具策略的优势：不同交易需求可以用不同的池子，避免"一刀切"。

## DeepBook（CLOB）

DeepBook 是 Sui 基金会支持的订单簿项目。它的设计充分利用了 Sui 的对象模型：

- 每个交易对是一个共享对象（OrderBook）
- 订单匹配在链上完成
- Maker 手续费低于 Taker（鼓励挂单）
- 适合需要精确价格的交易场景（如套利、做市）

## 选择框架

```
你需要什么？
│
├── 高资金效率、愿意主动管理 → Cetus (CLMM)
│
├── 简单交易、不想折腾 → Kriya (AMM 模式)
│
├── 大额交易、价格敏感 → DeepBook (CLOB)
│
├── 稳定币互换 → Kriya (Stableswap)
│
└── 做市、套利 → DeepBook + Cetus 组合
```

选择 DEX 不是选"最好的"，而是选"最适合你需求的"。每个 DEX 在特定场景下有优势。

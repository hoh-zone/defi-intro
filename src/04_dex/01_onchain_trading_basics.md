# 4.1 什么是链上交易

## Swap 的本质

在传统金融中，"交易"是买家和卖家通过中介（交易所、经纪商）完成资产互换。在 DeFi 中，"交易"（Swap）是用户与链上合约的交互——用户投入一种代币，合约返回另一种代币。

```
传统交易：
  买家 → 经纪商 → 交易所 → 卖家
  需要信任中介

链上 Swap：
  用户 → 智能合约（DEX）→ 自动执行
  不需要信任任何人（Trustless）
```

Swap 本质上是一个原子操作：要么完全成功（用户收到代币），要么完全失败（资金原路返回）。不存在"部分成交"或"交易失败但资金扣了"的情况。

## 三个核心角色

### LP（流动性提供者）

LP 将自己的资产存入 DEX 的流动性池中，为交易者提供可交易的资产。作为回报，LP 获得交易手续费的一部分。

```
LP 的行为：
  存入 1000 SUI + 2000 USDC 到 SUI/USDC 池
  获得 LP Token（代表池中的份额）
  每次有人交易，LP 获得手续费

LP 的风险：
  - 无常损失（Impermanent Loss，详见 4.12）
  - 价格风险（池中资产价格下跌）
  - 智能合约风险
```

### Trader（交易者）

Trader 使用 DEX 进行代币兑换。他们支付手续费，忍受滑点，获得即时流动性。

```
Trader 的行为：
  用 100 SUI 换 USDC
  DEX 根据当前池中比例计算可得 USDC 数量
  Trader 收到的 USDC 数量 = 池子定价 - 滑点 - 手续费

Trader 的关注点：
  - 价格是否合理（与外部市场价格对比）
  - 滑点大小（大额交易的冲击）
  - Gas 费用
```

### Arbitrageur（套利者）

套利者是 DEX 价格锚定外部市场的关键角色。当 DEX 价格与外部市场不一致时，套利者通过交易获利，同时将 DEX 价格拉回正确水平。

```
套利者的行为：
  外部市场 SUI = $2.00
  DEX 池中 SUI = $2.10（偏高）
  套利者：在 DEX 卖出 SUI → 赚取 $0.10/SUI 差价
  结果：DEX 中 SUI 供给增加，价格下降 → 趋近 $2.00

套利者的作用：
  ✅ 维持 DEX 价格与外部市场一致
  ✅ 为 LP 提供手续费收入
  ⚠️ 但也可能被三明治攻击利用（详见第 14 章）
```

## DEX 的基本功能模块

一个完整的 DEX 至少需要以下模块：

```
┌─────────────────────────────────────────┐
│              DEX 核心功能                 │
├─────────────────────────────────────────┤
│  Pool（流动性池）                        │
│    - 存储两种代币的储备量                 │
│    - 维护储备比例（决定价格）              │
├─────────────────────────────────────────┤
│  Swap Engine（交易引擎）                 │
│    - 根据储备量计算交易输出                │
│    - 更新储备量                          │
├─────────────────────────────────────────┤
│  LP Accounting（LP 会计）               │
│    - 记录每个 LP 的份额                   │
│    - 处理添加/移除流动性                   │
├─────────────────────────────────────────┤
│  Fee System（费用系统）                  │
│    - 收取交易手续费                       │
│    - 在 LP 和协议之间分配                  │
├─────────────────────────────────────────┤
│  Oracle（价格来源）                      │
│    - 输出当前池中代币价格                  │
│    - 供其他协议使用                       │
└─────────────────────────────────────────┘
```

## DEX 与 CEX 的根本差异

| 维度 | CEX（中心化交易所） | DEX（去中心化交易所） |
|------|-------------------|---------------------|
| 资产托管 | 交易所托管 | 用户自己控制 |
| 交易对手 | 订单簿撮合 | 与流动性池交互 |
| 价格发现 | 买卖挂单 | 算法定价（AMM）或订单簿 |
| 上币门槛 | 需要审核 | 任何人可创建池 |
| 抗审查 | 受监管 | 无需许可 |
| 速度 | 毫秒级 | 取决于区块链确认速度 |
| 前端运行 | 有监管 | MEV 是真实风险 |

理解这些差异，才能理解为什么 DEX 的设计选择与 CEX 截然不同——DEX 不是"链上的 CEX"，而是一种全新的交易范式。

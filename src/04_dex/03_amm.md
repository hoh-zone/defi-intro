# 4.3 AMM：恒定乘积做市商

## 核心问题

固定汇率 DEX 无法反映价格变化。我们需要一种算法，让价格**根据池子中两种代币的供需关系自动调整**。

## 恒定乘积公式

$$x \cdot y = k$$

- $x$ = 池中代币 A 的数量
- $y$ = 池中代币 B 的数量  
- $k$ = 常数（不考虑手续费时不变）

### 为什么这个公式能定价

当用户用 $\Delta x$ 的代币 A 换代币 B 时，池子必须保持 $k$ 不变：

$$(x + \Delta x) \cdot (y - \Delta y) = x \cdot y$$

解出 $\Delta y$：

$$\Delta y = y - \frac{x \cdot y}{x + \Delta x} = \frac{\Delta x \cdot y}{x + \Delta x}$$

**交易量越大，输出代币越少**——这就是滑点的来源。

### 价格 = 池内比例

池子的"边际价格"是两种代币的储量比：

$$P = \frac{y}{x}$$

当有人买入代币 B（x 增加，y 减少），P 上升。当有人卖出代币 B（x 减少，y 增加），P 下降。价格自动反映供需。

### 含手续费的输出

$$\Delta y_{out} = \frac{\Delta x \cdot (1 - f) \cdot y}{x + \Delta x \cdot (1 - f)}$$

其中 $f$ 是手续费率（如 0.3%）。

## 滑点

$$\text{Slippage} = \frac{P_{actual} - P_{expected}}{P_{expected}} = \frac{\Delta x}{x}$$

滑点正比于交易量占池子储备的比例。**池子越深，同等交易量的滑点越低。**

## 无常损失（Impermanent Loss）

LP 提供流动性后，如果池内价格发生变化，LP 取回的资产价值低于"持有不动"的价值。

$$IL = 2 \cdot \frac{\sqrt{P_1 / P_0}}{1 + P_1 / P_0} - 1$$

| 价格变化 | IL |
|----------|-----|
| 1.25x | -0.6% |
| 1.5x | -2.0% |
| 2.0x | -5.7% |
| 3.0x | -13.4% |
| 5.0x | -25.5% |

"无常"是因为价格回到初始值时损失消失。但如果价格永远不回来，损失就是永久的。

## 数值示例

初始状态：SUI/USDC 池

```
x (SUI)  = 1,000
y (USDC) = 2,000
k = 2,000,000
价格 = 2,000 / 1,000 = 2.0 USDC/SUI
```

用户 swap 100 USDC → SUI（费率 0.3%）：

```
fee = 100 × 0.003 = 0.3 USDC
amount_in_after_fee = 99.7 USDC

new_x = 1,000
new_y = 2,000 + 99.7 = 2,099.7
k不变? 不对——手续费使 k 增长

输出 SUI:
Δy_sui = 99.7 × 1,000 / (2,000 + 99.7) = 47.46 SUI

池子新状态:
x = 1,000 - 47.46 = 952.54 SUI
y = 2,000 + 100 = 2,100 USDC (手续费留在池子里)
新价格 = 2,100 / 952.54 = 2.205 USDC/SUI

实际成交价 = 100 / 47.46 = 2.108
滑点 = (2.108 - 2.0) / 2.0 = 5.4%
```

## AMM 的核心局限

| 局限 | 原因 | 影响 |
|------|------|------|
| 滑点大 | 价格与交易量非线性关系 | 大额交易成本高 |
| 资金效率低 | 流动性均匀分布在 (0, ∞) | 大部分资金浪费在极端价格 |
| 无常损失 | LP 被迫在价格变化时调整持仓 | LP 可能亏钱 |
| 报价不精确 | 价格是池内比例推导，不是真实供需 | 不适合价格敏感的交易 |

这些局限在后续的 Uniswap V3、DLMM 和订单簿中被逐步解决。

# 4.9 DEX 类型全景对比

## 六种 DEX 类型

从最简单到最复杂：

```
固定汇率 → StableSwap → AMM (V2) → V3 集中流动性 → DLMM → 订单簿
  ↑ 简单                                              复杂 ↑
  ↑ 低自由度                                          高自由度 ↑
```

## 全景对比

| 维度 | 固定汇率 | StableSwap | V2 AMM | V3 集中流动性 | DLMM | 订单簿 |
|------|----------|-----------|--------|-------------|------|--------|
| 价格机制 | 硬编码 | 曲线公式 | x·y=k | tick区间 | bin区间 | 限价撮合 |
| 滑点 | 零 | 极低 | 高 | 区间内低 | 区间内低 | 零（限价） |
| 资金效率 | 高 | 高 | 低 | 高 | 高 | 高 |
| 价格发现 | 无 | 有限 | 有 | 有 | 有 | 最精确 |
| LP 管理成本 | 无 | 无 | 无 | 高（手动调区间） | 中 | 高（挂单撤单） |
| 适用资产 | 同价资产 | 稳定币 | 任意 | 主流交易对 | 主流交易对 | 主流交易对 |
| 做市门槛 | 无 | 无 | 低 | 中 | 中 | 高 |
| Sui 代表 | — | Kriya | — | — | Cetus | DeepBook |

## 曲线形状对比

```
价格
  |      /
  |     / ← V2 AMM: x·y=k, 均匀弯曲
  |    /
  |   /
  |  /  ___
  | /  /   \  ← V3/DLMM: 在区间内平缓，区间外陡峭
  |/  /     \
  |--/-------\------- StableSwap: 几乎直线
  | /_________\______ 固定汇率: 完全直线
  |/_________________ 数量
```

## 选择框架

```
你要交易什么？
│
├── 稳定币对（USDC/USDT）→ StableSwap 或 固定汇率
│
├── 新代币（低流动性）→ V2 AMM（简单、低门槛）
│
├── 主流交易对（SUI/USDC）
│   ├── 想被动赚手续费 → DLMM (Cetus)
│   ├── 想主动管理区间 → V3 集中流动性
│   └── 想精确控制价格 → 订单簿 (DeepBook)
│
├── 大额交易 → 订单簿（限价单无滑点）
│
└── 做市/套利 → 订单簿 + DLMM 组合
```

## 每个 DEX 类型的数学核心

| DEX 类型 | 核心公式 | 变量 |
|----------|----------|------|
| 固定汇率 | output = input × rate | rate（固定） |
| StableSwap | A·n^n·Σx + D = A·D·n^n + D^(n+1)/(n^n·Πx) | A（放大系数） |
| V2 AMM | x·y = k | k（恒定乘积） |
| V3 集中流动性 | L = √(x·y)，在区间内 | L, tick_lower, tick_upper |
| DLMM | 每个 bin 独立定价 | bin_id, bin_step |
| 订单簿 | output = Σmin(order_size, remaining) at price_i | 限价单队列 |

## 在 Sui 上的实现选型

Sui 的对象模型对不同 DEX 类型有不同的优势：

| DEX 类型 | Sui 对象模型优势 |
|----------|-----------------|
| AMM/V2/V3/DLMM | 池子=共享对象，LP Token=拥有对象，天然分离 |
| StableSwap | 同上 |
| 订单簿 | **最大优势**：每个订单=独立对象，并行执行友好 |

DeepBook 选择在 Sui 上实现订单簿，正是因为 Sui 的对象模型让链上订单簿的可行性大幅提升。

# 《Sui DeFi 从入门到实战》出版级策划案

本仓库不是零散教程集合，而是按出版社审稿逻辑组织的一部中文技术书稿。

## 选题定位

- 书名建议：《Sui DeFi 从入门到实战》
- 副标题建议：从对象模型、协议机制到第一个可运行原型
- 核心卖点：不是只讲概念，也不是只抄协议介绍，而是把 Sui / Move / DeFi / 协议设计 / 风险边界 / 最小实战串成一条完整学习路径
- 目标读者：
  - 有编程基础，准备进入 Sui 生态的开发者
  - 想系统理解 DeFi 机制的产品经理、研究员、投资分析人员
  - 高校区块链课程或社团中需要中文系统教材的读者

## 出版社级别的成稿标准

如果要达到出版社愿意推进的标准，这本书至少要满足下面几个条件：

1. 目录稳定，章节边界清楚，不是想到哪里写到哪里。
2. 全书有明确阅读路径，从基础到协议再到工程化，而不是平铺协议介绍。
3. 每章都统一回答五个问题：
   - 这个协议赛道解决什么问题？
   - 资产如何流转？
   - 收益或费用从哪里来？
   - Sui 上为什么要这样设计？
   - 风险边界是什么？
4. 每章都要有固定交付物：
   - 本章目标
   - 先修知识
   - 机制图或流程图
   - Sui 案例
   - 关键结构体或伪代码
   - 风险提示
   - 小结与练习
5. 全书要有完整前置部分和附录：
   - 前言
   - 导读
   - 术语表
   - 公式与指标说明
   - 开发环境与命令清单
   - 延展阅读
6. 语言风格统一，避免教程口语、宣传口吻和投资建议表述。

## 建议成书规模

- 总字数：10.5 万到 11.5 万字
- 图表：40 到 55 张
- 代码或伪代码片段：30 到 45 段
- 核心样章：借贷篇中的 `Sui Savings`

## 目录重构结论

本次目录重构采用下面的原则：

- 用“编-章-节”三级结构，而不是只有章名
- 每个协议章都至少拆成 4 到 5 个小节，便于后续稳定写稿
- 单独加入“工程实践篇”，避免整本书只有赛道介绍，没有落地方法
- 把附录从正文中剥离出来，形成完整检索系统

## 第一个 Sui DeFi 项目怎么放

第一个完整实战项目确定为第 4 章第 4.1 节的 `Sui Savings`，定位如下：

- 类型：Sui 版固定利率储蓄池
- 功能：存钱、取钱、领取利息
- 教学目标：
  - 讲清楚池子和仓位对象
  - 讲清楚本金池与奖励池分离
  - 讲清楚 Sui 对象模型下的状态组织
  - 为后续借贷、CDP、清算打基础

第一版不引入下面这些复杂机制：

- 抵押借款
- 清算
- 预言机
- 动态利率
- 闪电贷

## 当前目录设计原则

1. 先理解 Sui 与 Move，再进入协议。
2. 先理解交易、价格、借贷这些基础设施，再看稳定币与收益层。
3. 先讲可运行的最小原型，再讲复杂金融结构。
4. 末章统一处理测试、部署和审计前自查，形成一本完整技术书。

## 当前阶段结论

这一轮不继续扩写正文，而是先把大纲固定到出版社可评估的级别。只要这套目录确认，后续每周都可以按章节推进，不会反复推翻结构。
